Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS08-062 - Ważna
======================================================

Luka w zabezpieczeniach usługi drukowania w Internecie systemu Windows umożliwia zdalne wykonanie kodu (953155)
---------------------------------------------------------------------------------------------------------------

Opublikowano: 14 października 2008 | Zaktualizowano: 29 października 2008

**Wersja:** 2.2

### Informacje ogólne

#### Streszczenie

Ta aktualizacja pozwala usunąć zgłoszoną przez osobę prywatną lukę w zabezpieczeniach usługi drukowania w Internecie systemu Windows, która może pozwolić na zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta.

Niniejsza aktualizacja ma wskaźnik ważności „ważny” dla wszystkich obsługiwanych wersji systemów Microsoft Windows 2000, Windows XP, Windows Server 2003 i Windows Server 2008. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa tę lukę, zmieniając sposób alokacji pamięci w protokole drukowania internetowego (IPP, Internet Printing Protocol). Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Firma Microsoft zaleca klientom jak najszybsze zastosowanie tej aktualizacji.

**Znane problemy.** Brak

Często zadawane pytania dotyczące tej aktualizacji zabezpieczeń
---------------------------------------------------------------

<span></span>
**Dlaczego 29 października 2008 r. zmieniono niniejszy biuletyn?**  
Firma Microsoft zaktualizowała niniejszy biuletyn w celu poinformowania klientów, że pakiety aktualizacyjne dla systemu Windows Server 2008 dla komputerów z procesorem Itanium i wszystkich obsługiwanych wersji systemu Windows Vista są już dostępne w usługach Windows Update, Microsoft Update, Windows Software Update Services (WSUS), programie Systems Management Server i programie System Center Configuration Manager. W chwili opublikowania tego biuletynu po raz pierwszy aktualizacje dla tych platform były dostępne tylko w witrynie Microsoft Download Center.

**Dlaczego 16 października 2008 r. zmieniono niniejszy biuletyn?**
Firma Microsoft zaktualizowała niniejszy biuletyn w celu wyjaśnienia, że usługa drukowania w Internecie systemu Windows działa w kontekście usługi buforu wydruku, która działa z uprawnieniami systemowymi. Osoba atakująca, której uda się wykorzystać lukę opisaną w tym biuletynie zabezpieczeń, może uruchomić dowolny kod z uprawnieniami systemowymi i uzyskać całkowitą kontrolę nad systemem podlegającym tej luce.

**Dlaczego 15 października 2008 r. zmieniono niniejszy biuletyn?**
Firma Microsoft zmieniła niniejszy biuletyn w celu usunięcia z niego wskaźnika ważności dla systemu Windows Server 2008 dla komputerów z procesorem Itanium. Firma Microsoft wprowadziła także modyfikacje mające na celu wyjaśnienie, że aktualizacja dla systemu Windows Server 2008 dla komputerów z procesorem Itanium jest dostępna w witrynie Microsoft Download Center i będzie publikowana w pozostałych kanałach dystrybucyjnych w najkrótszym możliwym terminie.

**Dlaczego system Windows Server 2008 dla komputerów z procesorem Itanium i system Windows** **Vista** **zostały umieszczone w tabeli „Programy, których dotyczy problem”, chociaż nie podano dla nich wskaźnika ważności?**
System Windows Server 2008 dla komputerów z procesorem Itanium i wszystkie obsługiwane wersje systemu Windows Vista zawierają wspólny składnik bufora wydruku, który jest aktualizowany przez tę aktualizację zabezpieczeń. Jednak ze względu na to, że system Windows Server 2008 dla komputerów z procesorem Itanium oraz system Windows Vista nie obsługują drukowania w Internecie, omawiany w niniejszym biuletynie zabezpieczeń potencjalny kierunek ataku nie jest otwarty. W ramach dodatkowego wzmocnienia ochrony przed ewentualnymi nowymi kierunkami ataku, które mogą zostać rozpoznane w przyszłości, zachęca się użytkowników systemu Windows Server 2008 dla komputerów z procesorem Itanium oraz systemu Windows Vista do zastosowania tej aktualizacji zabezpieczeń.

**Gdzie znajdują się szczegółowe informacje dotyczące plików?**  
Szczegółowe informacje dotyczące plików znajdują się w [artykule 953155 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/953155).

Luka w zabezpieczeniach usługi IPP związana z przepełnieniem liczby całkowitej — CVE-2008-1446  
----------------------------------------------------------------------------------------------
  
<span></span>
W systemach Windows z działającym programem Microsoft Internet Information Services (IIS) oraz włączoną usługą drukowania w Internecie systemu Windows istnieje luka, która umożliwia zdalne wykonanie kodu. Umożliwia ona zdalnej i uwierzytelnionej osobie atakującej wykonanie dowolnego kodu w systemie, którego dotyczy ten problem.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach usługi IPP związanej z przepełnieniem liczby całkowitej — CVE-2008-1446
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Usługa protokołu drukowania internetowego (IPP, Internet Printing Protocol) jest domyślnie włączona w programie IIS dla systemów Microsoft Windows 2000 i Windows XP. Jednak domyślne uprawnienia do plików na wszystkich platformach zostały skonfigurowane w sposób umożliwiający korzystanie z usługi IPP tylko użytkownikom z grupy „Uwierzytelnieni”. Ze względu na to ograniczenie użytkownicy nieuwierzytelnieni domyślnie nie mają uprawnień do folderów niezbędnych do zdalnego wykorzystania tej luki.  
-   Problem nie dotyczy domyślnie skonfigurowanych instalacji programu IIS w systemie Windows Server 2003. Zagrożone są tylko instalacje programu IIS w systemie Windows Server 2003 z włączonym filtrowaniem ISAPI protokołu IPP. Do obsługi zarządzania drukarkami sieci Web i drukowania w Internecie są wymagane składniki Drukowanie w Internecie oraz Strony ASP. Jeśli składnik Strony ASP znajduje się w grupie „Dozwolone”, a składnik Drukowanie w Internecie — w grupie „Zabronione”, zostanie włączona tylko usługa zarządzania drukarkami sieci Web. Użytkownicy nie będą mogli podłączyć się do żadnej drukarki udostępnionej za pośrednictwem usługi drukowania w Internecie.  
-   Domyślnie ten problem nie dotyczy instalacji programu IIS w systemie Windows Server 2008. Komputer z systemem Windows Server 2008 może działać jako komputer kliencki korzystający z usługi drukowania w Internecie. Użytkownicy komputera wysyłającego żądania wydruku muszą jednak zostać uwierzytelnieni przez serwer druku, aby móc korzystać z którejkolwiek drukarki podłączonej do tego serwera. Aby skonfigurować komputer z systemem Windows Server 2008 pełniącym rolę serwera z obsługą drukowania w Internecie, należy zainstalować rolę serwera sieci Web (IIS), rolę usług drukowania, a następnie rolę usług drukowania w Internecie w roli usług drukowania.  
-   Wykorzystanie tej luki wymaga komunikacji wychodzącej z serwera IIS do niebezpiecznego komputera za pośrednictwem portu 445. Administratorzy, którzy zablokują internetowe połączenia wychodzące protokołu SMB z serwera IIS mogą zapobiec wykorzystaniu tej luki z systemów niezaufanych.
  
#### Obejścia dotyczące luki w zabezpieczeniach usługi IPP związanej z przepełnieniem liczby całkowitej — CVE-2008-1446
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   **Wyłączenie usługi IPP**
  
    Aby wyłączyć usługę IPP w programach IIS 5.0 i IIS 5.1
  
    1.  Uruchom **Menedżera usług internetowych**.  
    2.  Kliknij prawym przyciskiem myszy **domyślną witrynę sieci Web** i wybierz polecenie **Właściwości**.  
    3.  Przejdź na kartę **Katalog macierzysty** i wybierz opcję **Konfiguracja**.  
    4.  Przewiń ekran w dół, aby odszukać rozszerzenie **printer** i odpowiednią ścieżkę do biblioteki **msw3prt.dll**.  
    5.  Kliknij przycisk **Usuń**.  
    6.  Kliknij **OK**.  
    7.  Ponownie uruchom usługę IIS.
  
    **Uwaga** Usługę drukowania w Internecie można skonfigurować zarówno przy użyciu zasad grupy, jak i Menedżera usług internetowych. W przypadku konfliktu między ustawieniami zasad grupy i Menedżera usług internetowych priorytet mają ustawienia zasad grupy. #### Często zadawane pytania dotyczące luki w zabezpieczeniach usługi IPP związanej z przepełnieniem liczby całkowitej — CVE-2008-1446
  
**Jaki zakres obejmuje ta luka?**    
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta.
  
**W jaki sposób powstaje luka?**    
Usługa protokołu drukowania internetowego (IPP) jest rozszerzeniem interfejsu ISAPI dla programu IIS. W implementacji usługi IPP firmy Microsoft na serwerach Windows z programem IIS istnieje luka umożliwiająca zdalne wykonanie kodu przez użytkowników uwierzytelnionych. W usłudze IPP istnieje luka przepełnienia liczby całkowitej umożliwiająca uwierzytelnionej osobie atakującej zdalne wykonanie kodu na serwerach IIS, których dotyczy ten problem.
  
**Co to jest ISAPI?**  
ISAPI (Internet Services Application Programming Interface) to technologia umożliwiająca programistom sieci Web zwiększenie funkcjonalności serwerów sieci Web przy użyciu własnego kodu, zapewniającego nowe usługi dla serwera sieci Web. Kod niestandardowy można zaimplementować w filtrze ISAPI, o ile nowa funkcja jest usługą niskiego poziomu, bądź w rozszerzeniu ISAPI, jeśli nowa funkcja jest usługą wysokiego poziomu. W tym przypadku kod, którego dotyczy problem, znajduje się w rozszerzeniu ISAPI.
  
**Co to jest rozszerzenie ISAPI, o którym wspomniano w tym biuletynie?**  
Rozszerzenie ISAPI, którego dotyczy problem, jest realizacją usługi IPP (Internet Printing Protocol) — standardu branżowego zdefiniowanego w specyfikacjach RFC 2910 i 2911. Usługa IPP zapewnia obsługę żądań usług drukowania i informacje na temat stanu zadań drukowania w Internecie za pośrednictwem protokołu HTTP. Na przykład użytkownik w podróży może przy użyciu usługi IPP wysłać zadanie drukowania za pośrednictwem Internetu do drukarki znajdującej się w jego sieci firmowej. Użytkownik może się również dowiedzieć, czy żądanie drukowania zostało ukończone bez błędów.
  
W systemie Microsoft Windows 2000 wprowadzono macierzystą obsługę drukowania w Internecie. W systemie Microsoft Windows 2000 można drukować bezpośrednio do adresu URL i wyświetlać informacje na temat zadań drukowania w przeglądarce. Obsługa drukowania w Internecie jest domyślnie włączona w systemach Microsoft Windows 2000 Server. Można ją również zainstalować jako składnik opcjonalny w systemach Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 i Windows Server 2008.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Osoba atakująca, której uda się wykorzystać tę lukę, może uruchomić dowolny kod w systemie użytkownika z uprawnieniami systemowymi. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może wysłać specjalnie spreparowane żądanie HTTP POST do serwera sieci Web, którego dotyczy problem, i spowodować, że serwer połączy się z komputerem udającym drukarkę IPP, kontrolowanym przez osobę atakującą. Następnie osoba atakująca może użyć specjalnie spreparowanych odpowiedzi, aby wykonać dowolny kod na serwerze docelowym.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Szczególnie zagrożone są stacje robocze oraz serwery z programem IIS i włączoną usługą IPP (Internet Printing Protocol) .
  
**W jaki sposób działa aktualizacja?**    
Niniejsza aktualizacja zabezpieczeń usuwa tę lukę, dodając procedurę sprawdzania poprawności danych otrzymanych od komputerów zdalnych w usłudze IPP.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**    
Tak. Firma Microsoft zna przypadki próby ataków wykorzystujących tę lukę. Jednak przed wydaniem niniejszego biuletynu zabezpieczeń przez firmę Microsoft nie opublikowano żadnych dowodów istnienia kodu koncepcyjnego.

-   **Narzędzie Microsoft Baseline Security Analyzer (MBSA)**

    Aby sprawdzić, czy ta aktualizacja zabezpieczeń została zainstalowana w systemie, którego dotyczy luka, można skorzystać z narzędzia Microsoft Baseline Security Analyzer (MBSA). -   **Weryfikacja wersji pliku**

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego w systemie, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

-   **Weryfikacja klucza rejestru**

    Pliki, które zostały zainstalowane przez tę aktualizację zabezpieczeń, można także zweryfikować, przeglądając klucze rejestru wymienione w niniejszej sekcji w **tabeli informacyjnej**.

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.
