Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS10-040 - Ważna
======================================================

Luka w zabezpieczeniach programu Internet Information Services umożliwia zdalne wykonanie kodu (982666)
-------------------------------------------------------------------------------------------------------

Opublikowano: 8 czerwca 2010 | Zaktualizowano: 30 czerwca 2010

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Ta aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach programu Internet Information Services (IIS), zgłoszoną przez użytkowników. Luka ta może pozwolić na zdalne wykonanie kodu, jeśli użytkownik otrzyma specjalnie spreparowane żądanie HTTP. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla programów IIS 6.0, IIS 7.0 i IIS 7.5. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę, zmieniając sposób sprawdzania poprawności uwierzytelniania na prawidłowy. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

**Znane problemy.** W [artykule 982666 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/982666) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń.

Często zadawane pytania dotyczące tej aktualizacji zabezpieczeń
---------------------------------------------------------------

<span></span>
**Jakich systemów dotyczy ta luka?**  
Luka dotyczy systemów Windows 7 i Windows Server 2008 R2. Oprócz tego dotyczy ona także systemów Windows Server 2003, Windows Vista i Windows Server 2008, w których zainstalowano niezwiązaną z zabezpieczeniami aktualizację [KB973917](http://support.microsoft.com/kb/973917) implementującą funkcję rozszerzonej ochrony uwierzytelniania.

**Czy mój komputer jest zagrożony, jeśli nie zainstalowano na nim aktualizacji KB973917?**  
Luka nie dotyczy systemów, w których zainstalowano obsługiwane wersje systemów Windows Server 2003, Windows Vista oraz Windows Server 2008 bez zainstalowanej aktualizacji KB973917. Systemy, w których zainstalowano obsługiwane wersje systemów Windows 7 i Windows Server 2008 R2, są zagrożone przez omawianą lukę.

**Czy otrzymam propozycję zainstalowania omawianej aktualizacji zabezpieczeń, jeśli w moim systemie nie zainstalowano aktualizacji KB973917?**  
Aktualizacja nie będzie proponowana w przypadku systemów, w których nie zainstalowano aktualizacji KB973917, ponieważ nie są one zagrożone przez omawianą lukę.

**Gdzie znajdują się szczegółowe informacje dotyczące plików?**  
Szczegółowe informacje dotyczące lokalizacji plików znajdują się w tabelach informacyjnych w sekcji **Wdrażanie aktualizacji zabezpieczeń**.

Luka w zabezpieczeniach mechanizmu uwierzytelniania programu IIS związana z uszkodzeniem pamięci — CVE-2010-1256  
----------------------------------------------------------------------------------------------------------------
  
<span></span>
W programie Internet Information Services (IIS) istnieje luka umożliwiająca zdalne wykonanie kodu. Przyczyną jej powstania jest nieprawidłowa analiza informacji dotyczących uwierzytelniania. Osoba atakująca, która wykorzystała tę lukę, może wykonać kod w kontekście procesu roboczego (WPI).
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2010-1256](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-1256).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach mechanizmu uwierzytelniania programu IIS związanej z uszkodzeniem pamięci — CVE-2010-1256
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Systemy Windows Server 2003, Windows Vista i Windows Server 2008 bez zainstalowanej aktualizacji KB973917 nie posiadają funkcji rozszerzonej ochrony uwierzytelniania i nie będą narażone na działanie omawianej luki.  
-   Funkcja rozszerzonej ochrony uwierzytelniania nie jest domyślnie włączona na żadnej z platform, których dotyczy luka, nawet gdy w systemie zainstalowano aktualizację KB973917. Luka stanowi zagrożenie dla tych systemów tylko, jeśli funkcja ta została włączona.
  
#### Obejścia problemu dotyczącego luki w zabezpieczeniach mechanizmu uwierzytelniania programu IIS związanej z uszkodzeniem pamięci — CVE-2010-1256
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   **Wymaganie sprawdzenia tokenu**
  
    Systemy, w których program Internet Information Services skonfigurowano do korzystania z funkcji rozszerzonej ochrony w celu obowiązkowego sprawdzania tokenów (zgodnie z opisem w [artykule 973917 bazy wiedzy Microsoft Knowledge Base)](http://support.microsoft.com/kb/973917) nie są narażone na omawianą lukę.
  
    **Wpływ obejścia.** Ta zmiana konfiguracji uniemożliwi uwierzytelnianie klientom bez zaimplementowanej funkcji rozszerzonej ochrony.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach mechanizmu uwierzytelniania programu IIS związanej z uszkodzeniem pamięci uwierzytelniania — CVE-2010-1256
  
**Jaki zakres obejmuje ta luka?**    
Jest to luka umożliwiająca zdalne wykonanie kodu. Zdalna, uwierzytelniona osoba atakująca, której uda się wykorzystać tę lukę, może wykonać w programie IIS Server działania z uprawnieniami procesu roboczego (WPI, Worker Process Identity). Ta tożsamość jest domyślnie skonfigurowana jako konto usługi sieciowej w systemach Windows Server 2003, Windows Vista oraz Windows Server 2008. W systemach Windows 7 i Windows Server 2008 R2 ta tożsamość jest skonfigurowana jako tożsamość puli aplikacji wirtualnych.
  
**W jaki sposób powstaje luka?**    
Przyczyną tej luki jest brak prawidłowej alokacji pamięci przez serwer Internet Information Services podczas analizowania określonych informacji dotyczących uwierzytelniania, otrzymanych od klienta.
  
**Jaki składnik jest dotknięty przez omawianą lukę?**    
Omawiana luka dotyczy serwera sieci Web Internet Information Services tylko wtedy, gdy jest on skonfigurowany do obsługi żądań uwierzytelniania przy użyciu funkcji rozszerzonej ochrony uwierzytelniania. Aby ten składnik był narażony na działanie luki, konieczne jest skonfigurowanie sprawdzania tokenów przez funkcję rozszerzonej ochrony poprzez zmianę ustawień na inne niż domyślne .
  
**Co to jest rozszerzona ochrona uwierzytelniania?**    
Rozszerzona ochrona uwierzytelniania to funkcja pomagająca chronić atakami typu man-in-the-middle (MITM), w których osoba atakująca przechwytuje poświadczenia klienta i przekazuje je do serwera. Funkcja rozszerzonej ochrony uwierzytelniania jest wdrażana wraz z aktualizacjami niezwiązanymi z zabezpieczeniami, które muszą zostać zainstalowane przez administratorów i użytkowników w celu zapewnienia dodatkowej ochrony.
  
Funkcja rozszerzonej ochrony uwierzytelniania jest dostępna jako aktualizacja niezwiązana z zabezpieczeniami dla systemów Windows Server 2003, Windows Vista i Windows Server 2008 (zobacz [artykuł 973917 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/973917)). W przypadku systemów Windows 7 i Windows Server 2008 R2 funkcja ta jest dostępna jako opcjonalny składnik systemu operacyjnego, który zostaje włączony przy instalacji usługi roli uwierzytelniania systemu Windows dla serwera Internet Information Services (IIS).
  
**Jak sprawdzić, czy w systemie włączono funkcję rozszerzonej ochrony uwierzytelniania?**    
Istnieje kilka sposobów skonfigurowania funkcji rozszerzonej ochrony uwierzytelniania. Informacje na temat konfigurowania tej funkcji, przydatne także do potwierdzenia jej obecności, można znaleźć w [artykule 973917 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/973917).
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Zdalna, uwierzytelniona osoba atakująca, której uda się wykorzystać tę lukę, może wykonać w programie IIS Server działania z uprawnieniami procesu roboczego (WPI, Worker Process Identity). Konto WPI domyślnie skonfigurowane jest z uprawnieniami konta usługi sieciowej. Serwery IIS ze stronami ASP, których pule aplikacji są skonfigurowane przy użyciu usługi korzystającej z konta z uprawnieniami administracyjnymi, mogą być bardziej narażone niż serwery IIS, których pula aplikacji jest skonfigurowana przy użyciu usługi WPI z ustawieniami domyślnymi.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
W serwerze IIS występuje luka umożliwiająca zdalne wykonanie kodu, związana ze sposobem przetwarzania prób uwierzytelnienia w sytuacji braku na kliencie funkcji rozszerzonej ochrony uwierzytelniania, która jest jednak włączona na serwerze. Osoba atakująca może wykorzystać tę lukę, tworząc specjalnie spreparowane żądanie HTTP.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Na działanie luki najbardziej narażone są serwery sieci Web obsługujące zawartość, która wymaga uwierzytelniania i obsługujące funkcję rozszerzonej ochrony uwierzytelniania. Systemy są narażone tylko wtedy, gdy zainstalowano w nich aktualizację implementującą funkcję rozszerzonej ochrony uwierzytelniania, która została opisana w [artykule 973917 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/973917).
  
**W jaki sposób działa aktualizacja?**    
Niniejsza aktualizacja usuwa lukę, poprawiając sposób, w jaki program IIS sprawdza poprawność określonych informacji dotyczących uwierzytelniania przekazywanych przez klienta.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**    
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.
