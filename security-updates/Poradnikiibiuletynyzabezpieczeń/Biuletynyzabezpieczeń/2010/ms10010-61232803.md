Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS10-010 - Ważna
======================================================

Luka w zabezpieczeniach funkcji Hyper-V systemu Windows Server 2008 umożliwia przeprowadzenie ataku typu „odmowa usługi” (977894)
---------------------------------------------------------------------------------------------------------------------------------

Opublikowano: 9 lutego 2010 | Zaktualizowano: 10 lutego 2010

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Ta aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach funkcji Hyper-V systemów Windows Server 2008 i Windows Server 2008 R2, która została zgłoszona przez użytkowników. Luka umożliwia przeprowadzenie ataku typu „odmowa usługi”, jeśli nieprawidłowa sekwencja instrukcji maszyny zostanie uruchomiona przez użytkownika uwierzytelnionego na jednej z maszyn wirtualnych gościa obsługiwanych przez serwer funkcji Hyper-V. Aby wykorzystać tę lukę, osoba atakująca musi dysponować prawidłowymi poświadczeniami logowania i być w stanie zalogować się lokalnie do maszyny wirtualnej gościa. Nie jest możliwe wykorzystanie luki w sposób zdalny lub przez użytkowników anonimowych.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla wszystkich obsługiwanych wersji systemów Microsoft Windows 2008 Server i Windows Server 2008 R2. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę, modyfikując sposób, w jaki serwer funkcji Hyper-V sprawdza poprawność kodu z instrukcjami maszyny wykonywanymi na jego maszynach wirtualnych gościa. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

**Co to jest serwer Hyper-V i czy jego również dotyczy ta luka?**

Hyper-V Server to dedykowany produkt autonomiczny, który zawiera tylko składnik Windows Hypervisor, model sterowników systemu Windows Server i składniki do wirtualizacji. Wymaga niewielkiej ilości zasobów przy minimalnych kosztach.
Ta luka dotyczy programów Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 z dodatkiem Service Pack 2 i Microsoft Hyper-V Server 2008 R2. Aby zapobiec jej wykorzystaniu, należy zastosować odpowiednio aktualizację dla systemu Windows Server 2008 lub Windows Server 2008 R2. Luka w zabezpieczeniach funkcji Hyper-V związana ze sprawdzaniem poprawności zestawu instrukcji - CVE-2010-0026  
---------------------------------------------------------------------------------------------------------------
  
<span></span>
Luka umożliwiająca przeprowadzenie ataku typu „odmowa usługi” istnieje w funkcji Hyper-V w systemach Windows Server 2008 i Windows Server 2008 R2. Przyczyną luki jest niedostateczne sprawdzanie poprawności określonych sekwencji instrukcji maszyny przez funkcję Hyper-V. Osoba atakująca, której uda się wykorzystać tę lukę, może spowodować, że zaatakowany system Hyper-V przestanie odpowiadać. Luka ta dotyczy wszystkich maszyn wirtualnych obsługiwanych przez ten system.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2010-0026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0026).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach funkcji Hyper-V związanej ze sprawdzaniem poprawności zestawu instrukcji - CVE-2010-0026
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Aby wykorzystać tę lukę, osoba atakująca musi dysponować prawidłowymi poświadczeniami logowania i być w stanie zalogować się lokalnie do maszyny wirtualnej Hyper-V. Nie jest możliwe wykorzystanie luki w sposób zdalny lub przez użytkowników anonimowych.
  
#### Obejścia problemu spowodowanego przez lukę w zabezpieczeniach funkcji Hyper-V związaną ze sprawdzaniem poprawności zestawu instrukcji - CVE-2010-0026
  
Firma Microsoft nie znalazła żadnego obejścia tej luki.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach funkcji Hyper-V związanej ze sprawdzaniem poprawności zestawu instrukcji - CVE-2010-0026
  
**Jaki zakres obejmuje ta luka?**  
Ta luka powoduje awarię typu odmowa usługi. Osoba atakująca, która wykorzysta tę lukę, może spowodować, że zaatakowany serwer Hyper-V przestanie odpowiadać i może być wymagane jego ponowne uruchomienie. Należy zauważyć, że usterka umożliwiająca atak typu „odmowa usługi" nie pozwala atakującemu na wykonanie kodu lub podniesienie poziomu swoich uprawnień, lecz może spowodować, że zaatakowany system przestanie akceptować żądania.
  
**W jaki sposób powstaje luka?**  
Przyczyną tej luki w serwerze Hyper-V jest nieprawidłowe sprawdzanie poprawności kodowania określonych instrukcji maszyny wykonywanych na maszynach wirtualnych gościa. W wyniku niedostatecznego sprawdzenia poprawności takich instrukcji ich przetworzenie może spowodować, że aplikacja serwera Hyper-V przestanie odpowiadać.
  
**Co to jest Hyper-V?**  
Hyper-V to technologia oparta na funkcji hypervisor, która jest kluczową funkcją systemów Windows Server 2008 i Windows Server 2008 R2. Zapewnia ona skalowalną, niezawodną i wysoce dostępną platformę wirtualizacji. Środowisko wirtualizacji jest przydatne, ponieważ użytkownicy mogą tworzyć maszyny wirtualne i zarządzać nimi, co pozwala im uruchamiać wiele systemów operacyjnych na jednym komputerze fizycznym przy jednoczesnym oddzieleniu tych systemów operacyjnych od siebie.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może spowodować, że system użytkownika przestanie odpowiadać aż do jego ponownego uruchomienia. Należy pamiętać, że wykorzystanie tej luki może spowodować, że może przestać odpowiadać rzeczywisty serwer Hyper-V, w tym wszystkie maszyny wirtualne gościa obsługiwane przez ten serwer.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Osoba atakująca musi być użytkownikiem uwierzytelnionym na jednej z maszyn wirtualnych gościa obsługiwanych przez serwer Hyper-V i mieć możliwość wykonania dowolnego kodu w systemie. Osoba atakująca może uruchomić niezaufany plik wykonywalny w systemie, który wywoła nieprawidłową sekwencję instrukcji maszyny i spowoduje, że serwer Hyper-V przestanie odpowiadać.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Szczególnie zagrożone są serwery Hyper-V obsługujące wirtualne maszyny gościa, które umożliwiają uzyskanie dostępu użytkownikom niezaufanym.
  
**W jaki sposób działa aktualizacja?**  
Aktualizacja usuwa lukę, modyfikując sposób, w jaki serwer funkcji Hyper-V sprawdza poprawność kodu z instrukcjami maszyny wykonywanymi na jego maszynach wirtualnych gościa.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**  
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**  
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.
