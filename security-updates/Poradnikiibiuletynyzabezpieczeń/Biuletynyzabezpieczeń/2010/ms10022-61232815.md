Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS10-022 - Ważna
======================================================

Luka w zabezpieczeniach aparatu obsługi skryptów VBScript umożliwia zdalne wykonanie kodu (981169)
--------------------------------------------------------------------------------------------------

Opublikowano: 13 kwietnia 2010 | Zaktualizowano: 5 maja 2010

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa zgłoszoną przez organizację publiczną lukę w zabezpieczeniach aparatu VBScript umożliwiająca zdalne wykonanie kodu. Niniejsza aktualizacja zabezpieczeń ma wskaźnik „ważny” dla systemów Microsoft Windows 2000, Windows XP i Windows Server 2003. W systemach Windows Server 2008, Windows Vista, Windows 7 i Windows Server 2008 R2 nie można wykorzystać kodu, którego dotyczy luka. Ponieważ kod ten jest obecny, niniejsza aktualizacja jest udostępniana jako dodatkowe wzmocnienie ochrony i nie ma dla niej wskaźnika ważności. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem, i programy, których nie dotyczy problem** niniejszej sekcji.

Luka ta może pozwolić na zdalne wykonanie kodu, gdy złośliwa witryna sieci Web wyświetli specjalnie spreparowane okno dialogowe na stronie sieci Web, a użytkownik naciśnie klawisz F1, powodując uruchomienie systemu pomocy Windows z użyciem pliku pomocy systemu Windows dostarczonego przez osobę atakującą. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca, która pomyślnie wykorzysta tę lukę, może uzyskać pełną kontrolę nad systemem, którego ta luka dotyczy.

Ta aktualizacja zabezpieczeń eliminuje lukę w zabezpieczeniach poprzez zmianę sposobu, w jaki aparat VBScript przetwarza pliki pomocy w trybie chronionym. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

Niniejsza aktualizacja zabezpieczeń dotyczy luki w zabezpieczeniach opisanej po raz pierwszy w [Poradniku zabezpieczeń firmy Microsoft 981169](http://technet.microsoft.com/security/advisory/981169).

**Zalecenie.**  Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi Microsoft Update.

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

**Znane problemy.** W [artykule 981169 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/981169) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń. Artykuł ten przedstawia również zalecane obejścia tych problemów.

Aby zapoznać się z zasadami cyklu pomocy technicznej dotyczącymi używanej wersji oprogramowania, odwiedź [witrynę zasad cyklu pomocy technicznej firmy Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Aby uzyskać więcej informacji na temat tej opcji instalacji, zobacz artykuły MSDN, [Server Core](http://msdn.microsoft.com/en-us/library/ms723891(vs.85).aspx) (j. ang.) i [Server Core for Windows Server 2008 R2](http://msdn.microsoft.com/en-us/library/ee391631(vs.85).aspx) (j. ang.). Należy pamiętać, że opcja instalacji Server Core nie dotyczy niektórych wersji systemu Windows Server 2008 i Windows Server 2008 R2. Zobacz [Porównanie opcji instalacji Server Core](http://www.microsoft.com/windowsserver2008/en/us/compare-core-installation.aspx).

Często zadawane pytania dotyczące tej aktualizacji zabezpieczeń
---------------------------------------------------------------

<span></span>
**Która wersja aparatu VBScript jest zainstalowana w systemie?**  
Wersja aparatu VBScript w systemie jest określana przez najnowszą wersję dołączoną do programu Internet Explorer, systemu operacyjnego Windows lub Hosta skryptów systemu Windows. Wersja aparatu może się zmienić w przypadku zainstalowania nowszej wersji programu Microsoft Internet Explorer, dodatku Windows Service Pack lub Hosta skryptów systemu Windows. Więcej informacji można znaleźć w artykule MSDN [Informacje o wersji aparatu VBScript](http://msdn.microsoft.com/en-us/library/4y5y7bh5(vs.85).aspx) (j. ang.).

Aby ustalić, która wersja aparatu VBScript jest zainstalowana w systemie, wykonaj następujące kroki:

1.  Otwórz program Eksplorator Windows.
2.  Przejdź do katalogu **%systemroot%\\system32**.
3.  Kliknij prawym przyciskiem plik **vbscript.dll**, wybierz polecenie **Właściwości**, a następnie kliknij kartę **Szczegóły**.

Numer wersji zostanie wyświetlony w polu **Wersja pliku**. Na przykład jeśli numer wersji pliku rozpoczyna się od 5.8 (na przykład 5.8.7600.16385), w systemie jest zainstalowany aparat VBScript 5.8.

**Używam aparatu VBScript 5.1. Czy luka dotyczy mojego systemu?**  
Tak. Aktualizacja zabezpieczeń KB981350 dotyczy danego systemu i spowoduje uaktualnienie aparatu VBScript do wersji VBScript 5.6. Więcej informacji na temat tej aktualizacji można znaleźć w [artykule 981350 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/981350).

**Uwaga** Aparat VBScript 5.1 jest obecny tylko w nowych instalacjach systemu Microsoft Windows 2000 z dodatkiem Service Pack 4, w którym nie zainstalowano jeszcze programu Internet Explorer 6. Zainstalowanie aktualizacji zabezpieczeń w takim systemie spowoduje uaktualnienie aparatu VBScript do wersji VBScript 5.6.

**Używam systemu Windows** **Vista, Windows Server 2008, Windows 7 lub Windows Server 2008 R2. Dlaczego otrzymuję tę aktualizację?**  
Luka w zabezpieczeniach istnieje w pliku systemu Windows, vbscript.dll, który znajduje się we wszystkich obsługiwanych wersjach systemów Windows Vista, Windows Server 2008, Windows 7 i Windows Server 2008 R2. Firma Microsoft nie znalazła żadnych sposobów na wykorzystanie tej luki na komputerach, na których używane są te systemy operacyjne. Niniejsza aktualizacja zabezpieczeń jest udostępniana jako dodatkowe wzmocnienie ochrony komputera wszystkim użytkownikom obsługiwanych wersji systemów Windows Vista, Windows Server 2008, Windows 7 i Windows Server 2008 R2.

**Jeśli zastosowano obejścia opisane w Poradniku 981169, czy należy je cofnąć przed zastosowaniem tej aktualizacji?**  
Jeżeli zastosowane zostały obejścia zawarte w [Poradniku zabezpieczeń firmy Microsoft 981169](http://technet.microsoft.com/security/advisory/981169), nie jest konieczne ich wycofanie przed zainstalowaniem tej aktualizacji. Aby wycofać te obejścia, można to bezpiecznie zrobić po zainstalowaniu aktualizacji. Poradnik wykorzystuje obejścia zalecane w tym biuletynie. Należy zapoznać się z odpowiednimi informacjami **Wpływ obejścia**, aby dowiedzieć się, co się stanie po zachowaniu obejścia.

Luka w zabezpieczeniach systemu pomocy aparatu VBScript związana z naciśnięciem klawisza — CVE-2010-0483
--------------------------------------------------------------------------------------------------------

<span></span>
W sposobie interakcji aparatu VBScript z plikami pomocy systemu Windows podczas korzystania z programu Internet Explorer występuje luka w zabezpieczeniach umożliwiająca zdalne wykonanie kodu. Jeśli złośliwa witryna sieci Web wyświetli specjalnie spreparowane okno dialogowe, a użytkownik naciśnie klawisz F1, zostanie uruchomiony system pomocy Windows z użyciem pliku pomocy systemu Windows dostarczonego przez osobę atakującą. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca, która pomyślnie wykorzysta tę lukę, może uzyskać pełną kontrolę nad systemem, którego ta luka dotyczy. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi. W systemach działających pod kontrolą systemu Windows Server 2003 domyślnie włączona jest [Konfiguracja zwiększonych zabezpieczeń](http://msdn.microsoft.com/en-us/library/ms537180(vs.85).aspx) programu Internet Explorer, co pomaga w ograniczeniu wpływu tego problemu.

Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2010-0483](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0483).

#### Czynniki ograniczające zagrożenie w przypadku luki w zabezpieczeniach systemu pomocy aparatu VBScript związanej z naciśnięciem klawisza — CVE-2010-0483

Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:

-   Tej luki nie da się wykorzystać w systemie Windows Vista, Windows Server 2008, Windows 7 ani Windows Server 2008 R2.
-   Luka wymaga działania ze strony użytkownika. Luka ta nie może zostać wykorzystana bez udziału użytkownika, nawet jeśli użytkownik odwiedzi niebezpieczną witrynę sieci Web. Osoba atakująca musiałaby przekonać użytkownika do naciśnięcia klawisza F1 na klawiaturze w momencie, gdy witryna sieci Web będzie wyświetlać zainicjowane przez skrypt okno dialogowe.
-   Osoba atakująca za pośrednictwem sieci Web, która chce wykorzystać tę lukę, może udostępniać witrynę zawierającą odpowiednią stronę sieci Web. Dodatkowo, witryny sieci Web, które akceptują lub udostępniają zawartość wprowadzoną przez użytkownika lub reklamy, jak również witryny niesprawdzone, mogą zawierać specjalnie spreparowane treści, które mogą wykorzystać lukę w zabezpieczeniach. Jednakże w żadnym przypadku osoba atakująca nie może zmusić użytkowników do odwiedzenia takich witryn. Musi natomiast przekonać użytkowników do odwiedzenia tej witryny, najczęściej zachęcając do kliknięcia łącza w wiadomości e-mail lub wiadomości błyskawicznej, które przekieruje użytkowników do witryny osoby atakującej.
-   W domyślnej konfiguracji wszystkie obsługiwane wersje programów Microsoft Outlook, Microsoft Outlook Express, i Poczta systemu Windows otwierają wiadomości e-mail w formacie HTML w strefie Witryny z ograniczeniami. Oznacza to, że osoba atakująca nie może wykorzystać tej luki w zabezpieczeniach do uruchomienia złośliwego kodu. Strefa Witryny z ograniczeniami umożliwia ograniczenie liczby ataków osób usiłujących wykorzystać tę lukę, zapobiegając wykonywaniu aktywnych skryptów i wykorzystywaniu formantów ActiveX do odczytywania wiadomości e-mail w formacie HTML. Jeśli jednak użytkownik kliknie łącze znajdujące się w wiadomości e-mail, może być nadal zagrożony wykorzystaniem tej luki poprzez atak za pośrednictwem sieci Web. Dodatkowo, program Outlook 2007 używa innego składnika do wyświetlania wiadomości e-mail. Oznacza to, że wykorzystanie tej luki nie będzie możliwe.
-   Domyślnie program Internet Explorer w systemie Windows Server 2003 jest uruchamiany w trybie chronionym określanym jako [Konfiguracja zwiększonych zabezpieczeń](http://go.microsoft.com/fwlink/?linkid=92039). W tym trybie poziom zabezpieczeń strefy Internet jest Wysoki. Jest to czynnik ograniczający zagrożenie w przypadku witryn sieci Web, których nie dodano do strefy witryn zaufanych programu Internet Explorer.
-   Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać takie same uprawnienia, jak użytkownik lokalny. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.

#### Obejścia dotyczące luki w zabezpieczeniach systemu pomocy aparatu VBScript związanej z naciśnięciem klawisza — CVE-2010-0483

Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:

-   **Nie należy naciskać klawisza F1, gdy prosi o to witryna sieci Web**

    Aby można było wykorzystać tę lukę, użytkownik musi ją „wspomóc”, naciskając klawisz F1 na klawiaturze. Przeprowadzone przez nas analizy dowodzą, że jeśli użytkownik nie naciśnie klawisza F1 na klawiaturze, luki nie uda się wykorzystać.

    Niebezpieczne witryny sieci Web mogą próbować przekonać użytkownika do naciśnięcia klawisza F1. Witryny takie mogą wyświetlać niekończącą się pętlę okien dialogowych, które informują użytkownika, że w celu zakończenia pętli powinien nacisnąć klawisz F1. Mogą też sugerować, że po naciśnięciu tego klawisza wyświetlone zostaną informacje o cenach albo pomoc.

#### Często zadawane pytania dotyczące luki w zabezpieczeniach systemu pomocy aparatu VBScript związanej z naciśnięciem klawisza — CVE-2010-0483

**Jaki zakres obejmuje ta luka?**  
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać takie same uprawnienia, jak zalogowany użytkownik. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.

**W jaki sposób powstaje luka?**  
Luka występuje w sposobie, w jaki VBScript współpracuje z plikami pomocy systemu podczas korzystania z programu Internet Explorer. Pliki pomocy systemu Windows są ze swojej natury niebezpiecznymi plikami, a implementacja aparatu VBScript w systemach, których dotyczy luka, umożliwia osobie atakującej określenie pliku pomocy podczas wyświetlania okna dialogowego na stronie sieci Web. Jeśli użytkownik naciśnie klawisz F1 podczas wyświetlania okna dialogowego, taki specjalnie spreparowany plik pomocy może zostać załadowany przez system pomocy Windows, umożliwiając wykonanie dowolnego kodu w kontekście zabezpieczeń aktualnie zalogowanego użytkownika.

**Co to jest VBScript?**  
VBScript (Visual Basic Script) to interpretowany, obiektowy język skryptów często używany do tworzenia witryn sieci Web w celu zapewnienia ich większej elastyczności lub interakcyjności. Skrypty VBScript można uruchomić tylko po zainstalowaniu interpretera lub hosta, takiego jak aktywne strony serwera (ASP, Active Server Pages), Internet Explorer lub Host skryptów systemu Windows. Więcej informacji można znaleźć w artykule MSDN [Podstawy języka VBScript](http://msdn.microsoft.com/en-us/library/0ad0dkea(vs.85).aspx) (j. ang.).

**Co to są niebezpieczne typy plików?**  
Firma Microsoft stosuje koncepcję „niebezpiecznych typów plików”, aby lepiej chronić użytkowników przed zawartością wykonywalną, która może zostać osadzona w takich plikach przez osobę atakującą. Więcej informacji na ten temat można znaleźć w oficjalnym dokumencie Zrozumienie zawartości wykonywalnej w produktach firmy Microsoft i w [artykule 291369 w bazie wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/291369).

**Co to jest klawisz F1?**  
F1 to jeden z dwunastu klawiszy funkcji (F1-F12) na typowej klawiaturze, zazwyczaj umieszczonych w górnym rzędzie klawiszy. Klawisze funkcji oferują użytkownikowi dostęp do pewnych funkcji aplikacji poprzez naciśnięcie jednego klawisza na klawiaturze. W większości aplikacji naciśnięcie klawisza F1 uruchamia funkcję pomocy. Wielu użytkowników doskonale zna takie polecenie, jak „Naciśnij klawisz F1, aby uzyskać pomoc”.

**Co to jest kompleksowe zabezpieczenie?**  
Określenie „kompleksowe zabezpieczenie” w dziedzinie bezpieczeństwa komputerowego oznacza podejście, w ramach którego stosowanych jest wiele warstw zabezpieczeń w celu powstrzymania osób atakujących przed naruszeniem zabezpieczeń sieci lub systemu.

**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może uruchomić dowolny kod jako użytkownik zalogowany. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca może uzyskać pełną kontrolę nad systemem podlegającym luce.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Aby można było wykorzystać tę lukę, osoba atakująca musi przekonać użytkownika do odwiedzenia specjalnie spreparowanej strony sieci Web, która wyświetla okno dialogowe, a następnie przekonać użytkownika do naciśnięcia klawisza F1 na klawiaturze podczas wyświetlania tego okna dialogowego.

**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Szczególnie narażone są systemy klientów, stacji roboczych oraz serwerów terminali, w których do przeglądania stron sieci Web służy program Internet Explorer.

**Używam programu Internet Explorer w systemie Windows Server 2003 lub Windows Server 2008. Czy jest to czynnik ograniczający zagrożenie wynikające z tej luki?**  
Domyślnie program Internet Explorer w systemach Windows Server 2003 i Windows Server 2008 jest uruchamiany w trybie chronionym określanym jako [Konfiguracja zwiększonych zabezpieczeń](http://go.microsoft.com/fwlink/?linkid=92039), który zmniejsza zagrożenie. Konfiguracja zwiększonych zabezpieczeń to grupa wstępnie skonfigurowanych ustawień programu Internet Explorer zmniejszających prawdopodobieństwo pobrania przez użytkownika lub administratora specjalnie spreparowanej zawartości z sieci Web na serwer i uruchomienia jej. Jest to czynnik ograniczający zagrożenie w przypadku witryn sieci Web, których nie dodano do strefy witryn zaufanych programu Internet Explorer. Zobacz także [Zarządzanie konfiguracją zwiększonych zabezpieczeń programu Internet Explorer](http://go.microsoft.com/fwlink/?linkid=92055).

**W jaki sposób ustawienie poziomu zabezpieczeń strefy Internet na Wysoki chroni użytkownika przed zagrożeniami związanymi z tą luką?**  
Ustawienie poziomu zabezpieczeń strefy Internet na Wysoki chroni przed działaniem luki, wyłączając wykonywanie aktywnych skryptów, które jest niezbędne w celu wykorzystania tej luki.

**Dlaczego tej luki w zabezpieczeniach nie można wykorzystać w systemach Windows** **Vista, Windows Server 2008, Windows 7 i Windows Server 2008 R2?**  
Systemy Windows Vista, Windows Server 2008, Windows7 i Windows Server 2008 R2 posiadają lepszą architekturę pod względem sposobu, w jaki program Internet Explorer prowadzi interakcję z systemem operacyjnym Windows i aparatem VBScript. Uniemożliwia to wykorzystanie tej luki w zabezpieczeniach pomimo tego, że niebezpieczny kod jest obecny w systemach, w których użytkownicy zainstalowali system pomocy Windows.

**W jaki sposób działa aktualizacja?**  
Ta aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach, modyfikując sposób, w jaki aparat VBScript przetwarza pliki pomocy w trybie chronionym w wersjach systemu operacyjnego Windows, którego dotyczy ta luka. W systemach Windows Vista, Windows Server 2008, Windows 7 i Windows Server 2008 R2 nie ma żadnych znanych kierunków ataków związanych z tą luką w zabezpieczeniach, ale można zainstalować tę aktualizację jako kompleksowe zabezpieczenie.

**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**  
Tak. Informacje o tej luce są znane publicznie. Została ona zarejestrowana w bazie danych znanych luk i zagrożeń pod numerem [CVE-2010-0483](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0483). Po raz pierwszy opisano ją w [Poradniku zabezpieczeń firmy Microsoft 981169](http://technet.microsoft.com/security/advisory/981169).

**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**  
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.

Podczas instalowania tej poprawki zabezpieczeń instalator sprawdza, czy dowolny z aktualizowanych plików na komputerze był wcześniej aktualizowany za pomocą poprawki hotfix firmy Microsoft.

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.

        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

-   **Narzędzie Microsoft Baseline Security Analyzer (MBSA)**

    Aby sprawdzić, czy ta aktualizacja zabezpieczeń została zainstalowana w systemie, którego dotyczy luka, można skorzystać z narzędzia Microsoft Baseline Security Analyzer (MBSA). -   **Weryfikacja wersji pliku**

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego w systemie, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

-   **Weryfikacja klucza rejestru**

    Pliki, które zostały zainstalowane przez tę aktualizację zabezpieczeń, można także zweryfikować, przeglądając klucze rejestru wymienione w niniejszej sekcji w **tabeli informacyjnej**.
