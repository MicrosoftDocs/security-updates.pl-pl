Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS10-101 - Ważna
======================================================

Luka w usłudze Netlogon w systemie Windows umożliwia przeprowadzenie ataku typu „odmowa usługi” (2207559)
---------------------------------------------------------------------------------------------------------

Opublikowano: 14 grudnia 2010

**Wersja:** 1.0

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa zgłoszoną przez użytkowników lukę w usłudze Netlogon RPC w zagrożonych wersjach systemu Windows Server skonfigurowanych jako kontrolery domen. Luka umożliwia osobie atakującej przeprowadzenie ataku typu „odmowa usługi”, jeśli osoba atakująca wyśle specjalnie spreparowany pakiet RPC do interfejsu usługi Netlogon RPC w zagrożonym systemie. W celu wykorzystania tej luki, osoba atakująca musi dysponować uprawnieniami administratora na komputerze podłączonym do tej samej domeny co kontroler domen zawierający tę lukę.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla wszystkich obsługiwanych wersji systemu Windows Server 2003, Windows Server 2008 (z wyjątkiem wersji dla systemów z procesorem Itanium) oraz Windows Server 2008 R2 (z wyjątkiem wersji dla systemów z procesorem Itanium). Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem, i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę, korygując sprawdzanie poprawności podawanych przez użytkownika danych w interfejsie usługi Netlogon RPC. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

Luka w zabezpieczeniach usługi Netlogon RPC związana z odwołaniem odniesienia do wskaźnika pustego i umożliwiająca atak typu „odmowa usługi” — CVE-2010-2742  
------------------------------------------------------------------------------------------------------------------------------------------------------------
  
<span></span>
W implementacji usługi Netlogon RPC w zagrożonych wersjach systemu Windows Server istnieje luka umożliwiająca zdalne przeprowadzenie ataku typu „odmowa usługi” przez użytkowników uwierzytelnionych. Po udanym wykorzystaniu luki przez osobę atakującą zagrożone wersje systemu Windows Server mogą zostać uruchomione ponownie.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2010-2742](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2742).
  
#### Czynniki ograniczające zagrożenie w przypadku luki w zabezpieczeniach usługi Netlogon RPC związanej z odwołaniem odniesienia do wskaźnika pustego i umożliwiającej atak typu „odmowa usługi” — CVE-2010-2742
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Luka dotyczy tylko serwerów z systemem Windows skonfigurowanych jako kontrolery domen.  
-   W celu wykorzystania tej luki, osoba atakująca musi dysponować uprawnieniami administratora na komputerze podłączonym do tej samej domeny co kontroler domen zawierający tę lukę.
  
#### Obejścia problemu związanego z luką w zabezpieczeniach usługi Netlogon RPC związaną z odwołaniem odniesienia do wskaźnika pustego i umożliwiającą atak typu „odmowa usługi” — CVE-2010-2742
  
Firma Microsoft nie znalazła żadnego obejścia tej luki.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach usługi Netlogon RPC związanej z odwołaniem odniesienia do wskaźnika pustego i umożliwiającej atak typu „odmowa usługi” — CVE-2010-2742
  
**Jaki zakres obejmuje luka?**    
Ta luka powoduje awarię typu odmowa usługi. Po wykorzystaniu luki przez osobę atakującą zaatakowany system mógłby zostać ponownie uruchomiony. Należy zauważyć, że luka umożliwiająca atak typu odmowa usługi nie pozwala atakującemu na wykonanie kodu lub podniesienie poziomu swoich uprawnień, lecz może spowodować ponowne uruchomienie się zaatakowanego systemu.
  
**Skąd bierze się luka?**    
Niewłaściwy sposób sprawdzania poprawności podawanych przez użytkownika danych w interfejsie usługi Netlogon RPC powoduje powstanie luki.
  
**Co to jest Netlogon RPC?**    
Protokół Netlogon Remote Protocol to interfejs zdalnego wywołania procedury (RPC) służący do uwierzytelniania komputerów i użytkowników w sieciach domenowych. Więcej informacji na temat usługi Netlogon RPC znajduje się w artykule MSDN [Netlogon Remote Protocol Specification](http://msdn.microsoft.com/en-us/library/cc237009(v=prot.10).aspx).
  
**Jakie mogą być skutki wykorzystania luki przez osobę atakującą?**     
Wykorzystanie tej luki mogłoby spowodować ponowne uruchomienie kontrolerów domen.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**     
Osoba atakująca może wykorzystać tę lukę, wysyłając specjalnie spreparowany pakiet RPC do interfejsu usługi Netlogon RPC. Na wszystkich zagrożonych platformach osoba atakująca musi posiadać prawidłowe poświadczenia uwierzytelniania i uprawnienia administratora, aby wykorzystanie luki było możliwe.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**     
Luka dotyczy serwerów z systemem Windows skonfigurowanych jako kontrolery domen z włączoną usługą Netlogon. Z tego względu systemy operacyjne Windows w wersjach klienckich nie są zagrożone tą luką.
  
**Jak mogę sprawdzić, czy mój system Windows Server jest skonfigurowany jako kontroler domeny?**    
Jeśli wartość klucza rejestru HKLM\\SYSTEM\\CurrentControlSet\\Control\\ProductOptions\\ProductType jest ustawiona na LanmanNT, system jest skonfigurowany jako kontroler domeny.
  
**W jaki sposób działa aktualizacja?**     
Aktualizacja usuwa lukę, korygując sprawdzanie poprawności podawanych przez użytkownika danych w interfejsie usługi Netlogon RPC.
  
**Czy luka była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od współpracujących źródeł.
  
**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.

**Instalowanie aktualizacji**
  
Podczas instalowania tej poprawki zabezpieczeń instalator sprawdza, czy dowolny z aktualizowanych plików na komputerze był wcześniej aktualizowany za pomocą poprawki hotfix firmy Microsoft.
