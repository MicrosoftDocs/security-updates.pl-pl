Szczegółowe informacje o luce w zabezpieczeniach
------------------------------------------------

<span></span>
#### Usterka usługi buforu wydruku — CAN-2005-1984:

W usłudze buforu wydruku występuje usterka umożliwiająca zdalne wykonanie kodu, przez co osoba atakująca, której uda się ją wykorzystać, może uzyskać pełną kontrolę nad systemem jej podlegającym.

#### Czynniki ograniczające zagrożenie spowodowane usterką usługi buforu wydruku — CAN-2005-1984:

-   Usterka dotyczy uwierzytelnionych użytkowników w systemach Windows XP z dodatkiem Service Pack 2 i Windows Server 2003. Aby ten problem umożliwił zdalne przeprowadzenie ataku w tych wersjach systemów operacyjnych, lokalny użytkownik posiadający odpowiednie uprawnienia musi dodatkowo udostępniać drukarkę albo próbować używać udostępnioną drukarkę. Jeśli żaden użytkownik posiadający wymagane uprawnienia nie udostępnia drukarki lub nie próbuje używać udostępnionej drukarki, osoba atakująca musiałaby posiadać prawidłowe poświadczenia logowania i mieć możliwość lokalnego zalogowania, by wykorzystać tę usterkę.
-   W wyniku tego problemu w systemach Microsoft Windows XP z dodatkiem Service Pack 1 i Windows Server 2003 powstaje usterka typu „odmowa usługi”. W systemach Windows XP z dodatkiem Service Pack 2 i Windows Server 2003 ten problem nie może być wykorzystany do zdalnego wykonania kodu ani podniesienia uprawnień.
    Próby ataków wykorzystujących tę lukę w innych wersjach systemów operacyjnych prawdopodobnie skończyłyby się sytuacją typu „odmowa usługi”. Zdalne wykonanie kodu może być jednak możliwe.
-   Sprawdzone metody działania stosowane w zaporach oraz standardowe domyślne konfiguracje zapór mogą pomóc w zabezpieczeniu sieci przed atakami spoza obszaru działania przedsiębiorstwa. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów.

#### Rozwiązania usterki buforu wydruku — CAN-2005-1984:

Firma Microsoft przetestowała następujące obejścia. Nie spowodują one wyeliminowania samej luki, przyczynią się jednak do zablokowania znanych kierunków ataków. W poniższej sekcji przedstawiono przypadki, w których zastosowanie obejścia zmniejsza funkcjonalność systemu.

-   **Wyłączenie usługi buforu wydruku**

    Wyłączenie usługi buforu wydruku może pomóc w zabezpieczeniu systemu podlegającego usterce przed próbami jej wykorzystania. Aby wyłączyć usługę buforu wydruku, należy wykonać następujące czynności:

    1.  Kliknij przycisk **Start**, a następnie polecenie **Panel sterowania** .Alternatywnie wskaż polecenie **Ustawienia**, a następnie kliknij polecenie **Panel sterowania**.
    2.  Kliknij dwukrotnie folder **Narzędzia administracyjne**.
    3.  Kliknij dwukrotnie ikonę **Usługi**.
    4.  Kliknij dwukrotnie ikonę **Bufor wydruku**.
    5.  Na liście **Typ uruchomienia** kliknij opcję **Wyłączony**.
    6.  Kliknij przycisk **Zatrzymaj**, a następnie przycisk **OK**.

    Usługę buforu wydruku można również zatrzymać i wyłączyć za pomocą następującego polecenia wykonywanego z wiersza poleceń:

    `sc stop Spooler & sc config Spooler start= disabled`

    **Wpływ obejścia:** Wyłączenie usługi buforu wydruku uniemożliwia drukowanie za pomocą drukarki lokalnej lub udostępnionej. Dlatego firma Microsoft zaleca to obejście tylko w przypadku systemów, które nie wymagają funkcji drukowania.

-   **W systemie Windows 2000 Server z zainstalowanym dodatkiem Service Pack 4 należy usunąć usługę buforu wydruku z klucza rejestru NullSessionPipes:**
    Systemy Windows 2000 Server z zainstalowanym dodatkiem Service Pack 4 i Windows XP z dodatkiem Service Pack 1 umożliwiają anonimowe połączenia z usługą, której dotyczy problem. Usunięcie usługi buforu wydruku z klucza rejestru NullSessionPipes ułatwia zapobieganie próbom wykorzystania tej usterki przez anonimowego intruza. To rozwiązanie nie zapobiega atakom ze strony uwierzytelnionych użytkowników. Opisanego rozwiązania należy użyć tylko w przypadku braku możliwości wyłączenia usługi buforu wydruku.

    **Uwaga** Nieprawidłowe korzystanie z Edytora rejestru może być przyczyną poważnych problemów, w wyniku których może być konieczna ponowna instalacja systemu operacyjnego. Firma Microsoft nie może zagwarantować rozwiązania problemów wynikających z nieprawidłowego użycia Edytora Rejestru. Zmian w rejestrze można dokonywać wyłącznie na własne ryzyko. Informacje na temat sposobu modyfikacji rejestru można znaleźć w temacie Pomocy „Zmienianie kluczy i wartości” programu Edytor rejestru (Regedit.exe) lub „Dodawanie i usuwanie informacji w rejestrze” oraz „Edytowanie danych rejestru” w przypadku programu Regedt32.exe.

    **Uwaga** Przed przystąpieniem do modyfikacji rejestru zaleca się utworzenie jego kopii zapasowej.

    1.  Kliknij przycisk **Start**, kliknij polecenie **Uruchom**, wpisz „**regedt32**” (bez cudzysłowów), a następnie kliknij przycisk **OK**.
    2.  W Edytorze rejestru odszukaj następujący klucz:

        **HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\LanmanServer\\Parameters\\NullSessionPipes**

    3.  Przeprowadź edycję tego klucza rejestru, usuwając wartość **SPOOLSS**.
    4.  Po wykonaniu powyższych czynności ponownie uruchom system podlegający usterce.

    **Wpływ obejścia:** Brak możliwości uzyskania anonimowego połączenia z usługą buforu wydruku. Jest to domyślne zachowanie nowszych wersji systemów operacyjnych.

#### Często zadawane pytania dotyczące usterki buforu wydruku — CAN-2005-1984:

**Jaki zakres obejmuje ta luka?**

Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może zdalnie uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Jednak próby wykorzystania tej usterki prawdopodobnie skończyłyby się sytuacją typu „odmowa usługi”.

**W jaki sposób powstaje luka?**
Niesprawdzony bufor w usłudze buforu wydruku.

**Co to jest usługa buforu wydruku?**
Usługa buforu wydruku, Spoolsv.exe, jest plikiem wykonywalnym zainstalowanym jako usługa. Bufor jest ładowany przy uruchomieniu systemu operacyjnego i pozostaje aktywny do jego wyłączenia. Usługa buforu wydruku zarządza procesem drukowania, który obejmuje zadania, takie jak: określenie lokalizacji odpowiedniego sterownika drukarki, ładowanie sterownika, umieszczanie w buforze wywołań funkcji wyższego poziomu dla zadania drukowania i tworzenie harmonogramu zadań drukowania. Gdy wszystkie odpowiednie czynności zostaną wykonane, zadanie drukowania jest przekazywane przez usługę buforu wydruku do routera drukowania. Więcej informacji na temat usługi buforu wydruku można znaleźć w następującej [witrynie sieci Web (j.ang.)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/techref/d58ce7b9-49cf-4f5e-95e9-1ade005c13e0.mspx).

**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**
Osoba atakująca, której uda się wykorzystać tę lukę umożliwiającą zdalne wykonanie kodu, może uzyskać pełną kontrolę nad systemem jej podlegającym. W wyniku tego problemu w systemach Microsoft Windows XP z dodatkiem Service Pack 2 i Windows Server 2003 powstaje błąd typu „odmowa usługi”. Próby ataków wykorzystujących tę usterkę w innych wersjach systemów operacyjnych prawdopodobnie skończyłyby się sytuacją typu „odmowa usługi”. Zdalne wykonanie kodu może być jednak możliwe.

**Kto może wykorzystać lukę?**
W systemach Windows 2000 i Windows XP z zainstalowanym dodatkiem Service Pack 1 każdy użytkownik anonimowy, mający możliwość dostarczenia do systemu podlegającego usterce specjalnie spreparowanego komunikatu, może spróbować wykorzystać tę lukę. Usterka dotyczy uwierzytelnionych użytkowników w systemach Windows XP z dodatkiem Service Pack 2 i Windows Server 2003. Uwierzytelniona osoba atakująca może mieć również możliwość lokalnego zalogowania się do systemu i spróbować wykorzystać tę usterkę we wszystkich wersjach systemów operacyjnych podlegających usterce.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**
Osoba atakująca może próbować wykorzystać tę lukę, tworząc specjalnie spreparowany komunikat i wysyłając go do systemu podlegającego luce. Komunikat może wymusić na systemie podlegającym usterce wykonanie kodu w wersjach systemu operacyjnego i konfiguracjach, które były narażone na ten typ ataku zdalnego. Systemy Windows 2000 i Microsoft Windows XP z dodatkiem Service Pack 1 w domyślnej konfiguracji są narażone na ten typ ataku zdalnego. Przeprowadzanie zdalnego ataku na systemy Windows XP z zainstalowanym dodatkiem SP2 lub Windows Server 2003 nie jest możliwe, chyba że użytkownik z odpowiednimi uprawnieniami udostępnia drukarkę lub próbuje używać udostępnionej drukarki.
Aby wykorzystać lokalnie tę lukę we wszystkich wersjach systemów operacyjnych, osoba atakująca musi najpierw zalogować się do systemu. Następnie musi uruchomić specjalny program umożliwiający wykorzystanie usterki.

J**akie systemy są szczególnie narażone na skutki tej luki?**
Najbardziej zagrożone tą luką są systemy Windows 2000 i Windows XP z zainstalowanym dodatkiem Service Pack 1. Systemy Microsoft Windows XP z dodatkiem Service Pack 1 i Windows Server 2003 są mniej narażone na skutki tej usterki dzięki występującym w nich czynnikom ograniczającym zagrożenie. Jednakże systemy działające jako serwery drukowania są szczególnie narażone na skutki tej usterki.

**Czy lukę można wykorzystać zdalnie za pomocą Internetu?**
Tak. Osoba atakująca może wykorzystać lukę za pośrednictwem Internetu. Wskazówki dotyczące zapór i standardowe domyślne konfiguracje zapór mogą pomóc w ochronie przed atakami z sieci Internet. Firma Microsoft udostępnia informacje mogące pomóc w zabezpieczeniu komputera. Użytkownicy końcowi mogą odwiedzić [witrynę sieci Web Chroń swój komputer](http://www.microsoft.com/poland/athome/security/protect/windowsxpsp2/default.mspx). Specjaliści IT mogą odwiedzić witrynę sieci Web [Security Guidance Center](http://www.microsoft.com/poland/technet/security/) (centrum porad w zakresie bezpieczeństwa).

**W jaki sposób działa aktualizacja?**
Aktualizacja usuwa lukę, zmieniając sposób sprawdzania przez usługę buforu wydruku poprawności długości komunikatu przed jej przekazaniem do przydzielonego buforu.

**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.

**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki w czasie, gdy opublikowano niniejszy biuletyn?**
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.
