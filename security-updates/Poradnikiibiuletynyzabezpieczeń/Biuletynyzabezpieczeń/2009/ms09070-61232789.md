Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS09-070 - Ważna
======================================================

Luki w zabezpieczeniach usługi Active Directory Federation Services umożliwiają zdalne wykonanie kodu (971726)
--------------------------------------------------------------------------------------------------------------

Opublikowano: 8 grudnia 2009 | Zaktualizowano: 9 grudnia 2009

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Ta aktualizacja zabezpieczeń usuwa dwie luki w zabezpieczeniach systemu Windows, które zostały zgłoszone przez użytkowników. Najpoważniejsza z tych luk umożliwia zdalne wykonanie kodu, jeśli osoba atakująca wyśle specjalnie spreparowane żądanie HTTP do serwera sieci Web z uruchomioną usługą ADFS. W celu wykorzystania którejkolwiek z tych luk osoba atakująca musi być użytkownikiem uwierzytelnionym.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „Ważny” dla wersji systemu Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 i Windows Server 2008 x64 Edition, których dotyczy luka. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa luki, modyfikując sprawdzanie poprawności nagłówków żądań przesłanych przez klientów sieci Web, które jest wykonywane przez serwery sieci Web z uruchomioną usługą ADFS. Więcej informacji na temat luk może znaleźć w podsekcjach „Często zadawane pytania” dla pozycji określonych luk w zabezpieczeniach, znajdujących się poniżej następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.**  Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

**Znane problemy.** [W artykule 971726 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/971726) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń. Artykuł ten przedstawia również zalecane obejścia tych problemów.

Ta aktualizacja obsługuje kilka luk, ponieważ modyfikacje konieczne do ich rozwiązania należy wprowadzić w powiązanych ze sobą plikach. Zamiast instalować klika niemal identycznych aktualizacji, klienci powinni skorzystać tylko z tej jednej.

**Co to jest usługa Active Directory Federation Services?**  
Usługa Active Directory Federation Services (ADFS) jest oparta na rozwijanej, branżowej architekturze usług sieci Web, która została zdefiniowana w specyfikacjach WS-\*. Usługa ADFS ułatwia korzystanie z mechanizmu rejestracji jednokrotnej (SSO) w celu uwierzytelnienia użytkowników w wielu powiązanych aplikacjach sieci Web, podczas całej pojedynczej sesji online. W tym celu usługa ADFS bezpiecznie udostępnia tożsamość cyfrową i upoważnienia w granicach zabezpieczeń i przedsiębiorstwa.

**Jak mogę sprawdzić, czy usługa ADFS jest zainstalowana na serwerze? **
Aby sprawdzić, czy usługa ADFS jest zainstalowana na serwerze, kliknij kolejno przycisk **Start**, polecenie **Panel sterowania**, polecenie **Dodaj/usuń programy**, kliknij dwukrotnie ikonę **Składniki systemu Windows**, a następnie wyszukaj usługę **Federation Services** na liście zainstalowanych składników.

**Co to jest Windows Server 2003 R2?**  
System Windows Server 2003 R2 to rozszerzenie systemu Windows Server 2003 zapewniające efektywniejszą kontrolę dostępu do lokalnych i zdalnych zasobów oraz zarządzanie nim, które można łatwo zintegrować z istniejącym środowiskiem systemu Windows Server 2003. System Windows Server 2003 R2 zapewnia skalowalną platformę sieci Web z dodatkowymi zabezpieczeniami, bezpośrednie współdziałanie z systemami opartymi na systemie UNIX i obsługę nowych scenariuszy, w tym uproszczonego zarządzania serwerami w oddziałach, udoskonalonego zarządzania tożsamością i dostępem oraz skuteczniejszego zarządzania pamięcią masową. Windows Server 2003 R2 Datacenter Edition i Windows Server 2003 R2 Enterprise Edition to również nowa dynamiczna usługa licencjonowania, która jeszcze bardziej zwiększa korzyść klientów z wirtualizacji serwerów.

Więcej informacji na temat systemu Windows Server 2003 R2 znajduje się w witrynie [Windows Server 2003 R2](http://www.microsoft.com/windowsserver2003/default.mspx).

**Co to są specyfikacje WS-\* sieci Web?**

Specyfikacje architektury usług sieci Web są oparte na standardach branżowych, takich jak SOAP (Simple Object Access Protocol), XML, WSDL (Web Service Description Language) i UDDI (Universal Description, Discovery, and Integration). Specyfikacja WS-\* to założenia dotyczące dostarczania pełnych, współdziałających ze sobą rozwiązań biznesowych dla rozbudowanego przedsiębiorstwa, w tym możliwości zarządzania tożsamością federacyjną i zabezpieczeniami.

Model usług sieci Web jest oparty na idei, w której systemy dla przedsiębiorstw są pisane w różnych językach, za pomocą różnych modeli programowania, które działają i są dostępne w urządzeniach różnego typu. Usługi sieci Web to środki do tworzenia systemów rozproszonych połączonych oraz współdziałających ze sobą łatwo i efektywnie za pośrednictwem Internetu, niezależnie od języka, w którym zostały napisane i na jakiej platformie są uruchomione.

Luka w zabezpieczeniach związana z fałszowaniem rejestracji jednokrotnej w usłudze ADFS – CVE-2009-2508  
-------------------------------------------------------------------------------------------------------
  
<span></span>
Luka w zabezpieczeniach usługi Active Directory Federation Services związana z fałszowaniem umożliwia osobie atakującej podszycie się pod użytkownika uwierzytelnionego, jeśli uzyska ona dostęp do stacji roboczej i przeglądarki sieci Web używanych ostatnio przez tego użytkownika w celu wyświetlenia witryny sieci Web z rejestracją jednokrotną.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach związanej z fałszowaniem rejestracji jednokrotnej w usłudze ADFS – CVE-2009-2508
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Osoba atakująca nie może wybrać użytkownika do podszycia się. Osoba atakująca może podszyć się wyłącznie pod użytkownika, którego token uwierzytelniania uda się jej przechwycić (na przykład w kiosku internetowym).  
-   Atak jest możliwy tylko przed wygaśnięciem ważności sesji na serwerze. Domyślnie sesje pozostają otwarte na serwerze przez 600 minut. Jeśli po upływie określonego limitu czasu osoba atakująca spróbuje ponownie użyć tego samego tokena uwierzytelniania, atak zakończy się niepowodzeniem.  
-   Osoba atakująca musi mieć dostęp do tokena uwierzytelniania, który został użyty w tym terminalu przez poprzedniego użytkownika. W przypadku serwera z obsługą HTTPS przesyłany token uwierzytelniania jest chroniony i w celu wykorzystania luki osoba atakująca musi uzyskać dostęp do komputera atakowanego użytkownika (na przykład w kiosku internetowym). W kiosku internetowym użytkownik może zarejestrować się w aplikacji sieci Web z rejestracją jednokrotną w usłudze ADFS, a następnie wyrejestrować się z aplikacji. Osoba atakująca może użyć tego komputera, aby uzyskać dostęp do aplikacji sieci Web jako poprzednio zarejestrowany użytkownik, nawet jeśli użytkownik ten wylogował się.
  
#### Obejścia luki w zabezpieczeniach związanej z fałszowaniem rejestracji jednokrotnej w usłudze ADFS – CVE-2009-2508
  
Firma Microsoft nie znalazła żadnego obejścia tej luki.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach związanej z fałszowaniem rejestracji jednokrotnej w usłudze ADFS – CVE-2009-2508
  
**Jaki zakres obejmuje ta luka?**    
Jest to luka w zabezpieczeniach związana z fałszowaniem. Osoba atakująca, której uda się wykorzystać tę lukę, może podszyć się pod innego użytkownika. Dzięki temu będzie ona mogła uzyskać dostęp jako zaatakowany użytkownik do witryny sieci Web z rejestracją jednokrotną usługi ADFS.
  
**W jaki sposób powstaje luka?**    
Przyczyną tej luki jest niewystarczające sprawdzanie poprawności zarządzania sesją w usłudze ADFS.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
W systemach z dostępem współdzielonym, takich jak kioski internetowe, osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać dostęp do zasobów należących do innego, poprzednio uwierzytelnionego użytkownika. Atak ten może zostać przeprowadzony tylko przed upływem limit czasu sesji, który jest konfigurowany przez administratora usługi ADFS. Domyślnie jest to 600 minut.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może wykorzystać tę lukę, uzyskując dostęp do określonych danych z pamięci podręcznej przeglądarki i ponownie używając ich w celu uwierzytelnienia się w aplikacji sieci Web, która korzysta z rejestracji jednokrotnej usługi ADFS. Po upływie limitu czasu na serwerze usługa ADFS zakończy uwierzytelnioną sesję po stronie serwera i osoba atakująca nie będzie już mogła ponownie użyć tych poświadczeń.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Ta luka dotyczy serwerów sieci Web korzystających z usługi ADFS do rejestracji jednokrotnej. Luka ta nie stanowi bezpośredniego zagrożenia dla systemów klienckich.
  
**W jaki sposób działa aktualizacja?**    
Aktualizacja usuwa lukę, powodując, że poświadczenia uwierzytelniania użytkownika, który wylogował się z sieci, będą prawidłowo odrzucane.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika.

Luka w zabezpieczeniach usługi ADFS umożliwiająca zdalne wykonanie kodu – CVE-2009-2509  
---------------------------------------------------------------------------------------
  
<span></span>
We implementacjach usługi ADFS (Active Directory Federation Services) istnieje luka w zabezpieczeniach, która umożliwia zdalne wykonanie kodu. Luka wynika z nieprawidłowego sprawdzania poprawności nagłówków żądań podczas łączenia się użytkownika uwierzytelnionego z serwerem sieci Web korzystającym z usługi ADFS. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach usługi ADFS umożliwiającej zdalne wykonanie kodu – CVE-2009-2509
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Osoba atakująca musi posiadać ważne poświadczenia logowania do serwera, którego dotyczy luka. Atak może zostać przeprowadzony tylko po pomyślnym uwierzytelnieniu na serwerze korzystającym z usługi ADFS.
  
#### Obejścia luki w zabezpieczeniach usługi ADFS umożliwiającej zdalne wykonanie kodu – CVE-2009-2509
  
Firma Microsoft nie znalazła żadnego obejścia tej luki.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach usługi ADFS umożliwiającej zdalne wykonanie kodu – CVE-2009-2509
  
**Jaki zakres obejmuje ta luka?**    
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może wykonać w programie IIS Server działania z uprawnieniami procesu roboczego (WPI, Worker Process Identity), którego konto ma domyślnie uprawnienia konta usługi sieciowej.
  
**W jaki sposób powstaje luka?**    
Luka ta wynika z nieprawidłowego sprawdzania w usłudze ADFS poprawności nagłówków żądań, które są przesyłane przez uwierzytelnionych klientów.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Osoba atakująca, której uda się wykorzystać tę lukę, może wykonać w programie IIS Server działania z uprawnieniami procesu roboczego (WPI, Worker Process Identity). Konto WPI domyślnie skonfigurowane jest z uprawnieniami konta usługi sieciowej. Serwery IIS ze stronami ASP, których pule aplikacji są skonfigurowane przy użyciu usługi korzystającej z konta z uprawnieniami administracyjnymi, mogą być bardziej narażone niż serwery IIS, których pula aplikacji jest skonfigurowana przy użyciu usługi WPI z ustawieniami domyślnymi.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może próbować wykorzystać tę lukę, wysyłając specjalnie spreparowane żądanie HTTP do serwera sieci Web korzystającego z usługi ADFS, którego dotyczy luka. W celu wykorzystania tej luki osoba atakująca musi być użytkownikiem uwierzytelnionym.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Narażone są serwery sieci Web korzystające z usługi ADFS. Luka ta nie dotyczy systemów klienckich.
  
**W jaki sposób działa aktualizacja?**    
Niniejsza aktualizacja modyfikuje sprawdzanie poprawności nagłówków żądań, które są przesyłane przez klienta sieci Web, wykonywane na serwerach sieci Web korzystających z usługi ADFS.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika.

**Weryfikacja instalacji aktualizacji**

-   **Narzędzie Microsoft Baseline Security Analyzer (MBSA)**

    Aby sprawdzić, czy ta aktualizacja zabezpieczeń została zainstalowana w systemie, którego dotyczy luka, można skorzystać z narzędzia Microsoft Baseline Security Analyzer (MBSA). Więcej informacji na ten temat znajduje się w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się powyżej w niniejszym biuletynie.

-   **Weryfikacja wersji pliku**

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego w systemie, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

-   **Weryfikacja klucza rejestru**

    Pliki, które zostały zainstalowane przez tę aktualizację zabezpieczeń, można także zweryfikować, przeglądając klucze rejestru wymienione w niniejszej sekcji w **tabeli informacyjnej**.

    Te klucze rejestru mogą nie zawierać pełnej listy zainstalowanych plików. Klucze te mogą także nie zostać prawidłowo utworzone, jeśli administrator lub producent OEM zintegruje tę aktualizację zabezpieczeń ze źródłowymi plikami instalacyjnymi systemu Windows lub ją do nich włączy.
