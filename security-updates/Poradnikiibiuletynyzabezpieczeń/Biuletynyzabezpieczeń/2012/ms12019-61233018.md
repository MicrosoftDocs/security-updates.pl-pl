Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS12-019 - Umiarkowana
============================================================

Luka w zabezpieczeniach programu DirectWrite umożliwia przeprowadzenie ataku typu „odmowa usługi” (2665364)
-----------------------------------------------------------------------------------------------------------

Opublikowano: 13 marca 2012

**Wersja:** 1.0

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń ma na celu usunięcie zgłoszonej przez organizację publiczną luki w zabezpieczeniach programu Windows DirectWrite. W scenariuszu ataku z wykorzystaniem wiadomości błyskawicznych luka umożliwia atak typu „odmowa usługi”, jeśli osoba atakująca wyśle specjalnie spreparowaną sekwencję znaków Unicode bezpośrednio do klienta wiadomości błyskawicznych. Aplikacja docelowa może przestać reagować podczas próby renderowania specjalnie spreparowanej sekwencji znaków Unicode przez program DirectWrite.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „umiarkowany” dla wszystkich obsługiwanych wersji systemu Windows Vista, Windows Server 2008 (z wyjątkiem Windows Server 2008 dla systemów z procesorem Itanium), Windows 7 i Windows Server 2008 R2. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę, zmieniając sposób, w jaki program DirectWrite renderuje znaki Unicode. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

Luka w zabezpieczeniach aplikacji DirectWrite umożliwiająca przeprowadzenie ataku typu „odmowa usługi” — CVE-2012-0156  
----------------------------------------------------------------------------------------------------------------------
  
<span></span>
W sposobie renderowania specjalnie spreparowanych sekwencji znaków Unicode przez program DirectWrite istnieje luka w zabezpieczeniach umożliwiająca przeprowadzenie ataku typu „odmowa usługi”. Po wykorzystaniu tej luki osoba atakująca może spowodować, że zagrożona aplikacja przestanie odpowiadać.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2012-0156](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0156).

Firma Microsoft nie znalazła dla tej luki żadnych czynników ograniczających zagrożenie.
  
#### Obejścia problemu spowodowanego przez lukę w zabezpieczeniach aplikacji DirectWrite umożliwiającą przeprowadzenie ataku typu „odmowa usługi” — CVE-2012-0156

#### Często zadawane pytania dotyczące luki w zabezpieczeniach aplikacji DirectWrite umożliwiającej przeprowadzenie ataku typu „odmowa usługi” — CVE-2012-0156
  
**Jaki zakres obejmuje ta luka?**  
Ta luka powoduje awarię typu odmowa usługi.
  
**W jaki sposób powstaje luka?**  
Luka powstaje, gdy program DirectWrite nieprawidłowo renderuje specjalnie spreparowaną sekwencję znaków Unicode w pamięci.
  
**Co to jest** **DirectWrite?**  
DirectWrite to interfejs API przeznaczony do renderowania tekstu, działający niezależnie od rozdzielczości i obsługujący cały system znaków Unicode, który wspomaga mierzenie, wyświetlanie, formatowanie i testowanie tekstu. Aby uzyskać więcej informacji na temat programu DirectWrite, zobacz artykuł MSDN [Introducing DirectWrite](http://msdn.microsoft.com/en-us/library/windows/desktop/dd371554(v=vs.85).aspx).
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Po wykorzystaniu tej luki osoba atakująca może spowodować, że zagrożona aplikacja (np. Windows Live Messenger) przestanie odpowiadać.
  
**W jaki sposób osoba atakująca może wykorzystać** **tę lukę?**  
Nieuwierzytelniona osoba atakująca za pośrednictwem sieci Web musiałaby udostępnić witrynę sieci Web zawierającą specjalnie spreparowaną sekwencję znaków Unicode. Dodatkowo witryny sieci Web, które akceptują lub udostępniają zawartość wprowadzoną przez użytkownika, jak również witryny, do których się włamano, mogą zawierać specjalnie spreparowane treści, które mogą posłużyć do wykorzystania luki w zabezpieczeniach. Aby istniała możliwość wykorzystania tej luki w zabezpieczeniach, specjalnie spreparowana sekwencja znaków Unicode musi zostać przetworzona przez aplikację, która renderuje zawartość stron sieci Web za pomocą programu DirectWrite. Zagrożona aplikacja może przestać reagować podczas próby renderowania specjalnie spreparowanej sekwencji znaków Unicode przez program DirectWrite. Osoba atakująca nie może w żaden sposób zmusić użytkowników do odwiedzenia witryny, na której umieszczono specjalnie spreparowaną sekwencję znaków Unicode. Musi przekonać użytkowników do odwiedzenia takiej witryny, najczęściej zachęcając do kliknięcia łącza w wiadomości e-mail lub wiadomości błyskawicznej, które przekieruje ich do witryny osoby atakującej.
  
W scenariuszu ataku z wykorzystaniem wiadomości błyskawicznych nieuwierzytelniona osoba atakująca może wykorzystać tę lukę w zabezpieczeniach, wysyłając specjalnie spreparowaną sekwencję znaków Unicode bezpośrednio do klienta wiadomości błyskawicznych, np. programu Windows Live Messenger. Aplikacja docelowa może przestać reagować podczas próby renderowania specjalnie spreparowanej sekwencji znaków Unicode przez program DirectWrite.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Zagrożone są zarówno stacje robocze, jak i serwery terminalowe. Zagrożenie serwerów może być większe, jeśli administratorzy zezwolą użytkownikom na logowanie do tych serwerów i uruchamianie programów. Jednak zestawy najlepszych praktyk zdecydowanie to odradzają.
  
**W jaki sposób działa aktualizacja?**  
Aktualizacja usuwa lukę, zmieniając sposób, w jaki program DirectWrite renderuje znaki Unicode.
  
**Czy luka była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**  
Tak. Informacje o tej luce są znane publicznie. Do luki został przypisany numer w bazie danych znanych luk i zagrożeń [CVE-2012-0156](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0156).
  
**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki w czasie, gdy opublikowano niniejszy biuletyn?**  
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.

Zarządzanie oprogramowaniem oraz aktualizacjami zabezpieczeń, które należy zainstalować na serwerach oraz komputerach stacjonarnych i systemach przenośnych w organizacji. Więcej informacji można znaleźć w [Centrum TechNet Update Management Center](http://go.microsoft.com/fwlink/?linkid=69903). Witryna sieci Web [Microsoft TechNet Security](http://go.microsoft.com/fwlink/?linkid=21132) zawiera dodatkowe informacje na temat zabezpieczeń w produktach firmy Microsoft.
  
Aktualizacje zabezpieczeń dostępne są w witrynach [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) i [](http://go.microsoft.com/fwlink/?linkid=21130)[Windows Update](http://go.microsoft.com/fwlink/?linkid=21130)</a>. Aktualizacje zabezpieczeń są także dostępne w witrynie [Centrum pobierania Microsoft](http://go.microsoft.com/fwlink/?linkid=21129). Najłatwiej je znaleźć, wyszukując wyrażenie „aktualizacja zabezpieczeń”.

Wykaz usługi Microsoft Update zawiera katalog zawartości z możliwością przeszukiwania, który udostępniany jest poprzez usługi Windows Update i Microsoft Update i obejmuje aktualizacje zabezpieczeń, sterowniki i dodatki Service Pack. Wyszukiwanie przy użyciu numeru biuletynu zabezpieczeń (np. „MS07-036”) pozwala dodać do koszyka wszystkie odpowiednie aktualizacje (w tym różne wersje językowe aktualizacji) i pobrać pliki do wybranego folderu. Więcej informacji na temat Wykazu usługi Microsoft Update można znaleźć w [Często zadawanych pytaniach dotyczących Wykazu usługi Microsoft Update](http://go.microsoft.com/fwlink/?linkid=97900).

#### Microsoft Active Protections Program (MAPP)
  
W celu zwiększenia poziomu bezpieczeństwa swoich klientów firma Microsoft dostarcza informacje na temat luk w zabezpieczeniach największym dostawcom oprogramowania zabezpieczającego przed publikacją comiesięcznej aktualizacji zabezpieczeń. Dzięki informacjom dotyczącym luk w zabezpieczeniach dostawcy oprogramowania zabezpieczającego mogą zaoferować swoim klientom zaktualizowane poprawki za pośrednictwem programów lub urządzeń zabezpieczających np. programów antywirusowych, sieciowych systemów wykrywania włamań lub hostowych systemów zapobiegania włamaniom. Aby sprawdzić, czy dostawcy oprogramowania zabezpieczającego zapewniają aktywną ochronę, odwiedź witryny poszczególnych partnerów programu, których listę znaleźć można w sekcji [Microsoft Active Protections Program (MAPP) Partners](http://go.microsoft.com/fwlink/?linkid=215201).
