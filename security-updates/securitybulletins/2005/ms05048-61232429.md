Szczegółowe informacje o usterce  
--------------------------------
  
#### Usterka w obiektach Collaboration Data Objects (CDO) - CAN-2005-1987
  
W obiektach Collaboration Data Objects (CDO) występuje usterka umożliwiająca zdalne wykonanie kodu, przez co osoba atakująca, której uda się ją wykorzystać, może uzyskać pełną kontrolę nad systemem jej podlegającym.
  
#### Czynniki ograniczające zagrożenie w przypadku usterki obiektów Collaboration Data Objects (CDO) - CAN-2005-1987:
  
-   W konfiguracji domyślnej protokół SMTP (Simple Mail Transfer Protocol) usługi Microsoft Internet Information Services (IIS) 5.0 i usługa SMTP programu Exchange 2000 Server nie używają obiektów event sink wykorzystujących pliki Cdosys.dll i Codex.dll.  
-   Usługa IIS 6.0 nie jest domyślnie włączona w systemie Windows Server 2003.  
-   Usługa IIS 6.0, gdy jest włączona, nie włącza domyślnie protokołu SMTP.
  
#### Rozwiązania problemu spowodowanego usterką obiektów Collaboration Data Objects (CDO) - CAN-2005-1987:
  
Firma Microsoft przetestowała następujące rozwiązania. Nie spowodują one wyeliminowania samej usterki, przyczynią się jednak do zablokowania znanych kierunków ataków. W poniższej sekcji wskazano przypadki, w których rozwiązanie ogranicza funkcjonalność systemu.
  
-   **Wyłączenie wszystkich obiektów event sink włączonych na serwerach Exchange 2000 Server i na serwerach z uruchomioną usługą IIS:**
  
    Wyłączenie wszystkich ujść zdarzeń pomoże ochronić system podlegający usterce przed próbami jej wykorzystania. Aby wyłączyć wszystkie niestandardowe lub utworzone przez strony trzecie obiekty event sink, należy wykonać następujące czynności:
  
    1.  Wykonaj następujące polecenia **cscript.exe smtpreg.vbs /enum** z wiersza poleceń, aby wyświetlić wszystkie zarejestrowane obiekty event sink. Przejrzyj wyniki wykonania tego polecenia, szukając niestandardowych lub nieutworzonych przez oprogramowanie firmy Microsoft wpisów obiektów event sink.  
    2.  Po zidentyfikowaniu obiektu event sink, który chcesz wyłączyć, wykonaj następujące polecenie z wiersza poleceń:  
        **cscript.exe smtpreg.vbs /disable**
  
    **Wpływ rozwiązania:** Aplikacja niestandardowa lub innej firmy, która utworzyła obiekt event sink w usłudze SMTP, nie będzie działać prawidłowo do momentu ponownego włączenia tego obiektu.
  
-   **Wyrejestrowanie plików Cdoex.dll i Cdosys.dll na serwerze Exchange 2000 Server oraz pliku Cdosys.dll na serwerach z uruchomioną usługą IIS:**
  
    Wyrejestrowanie plików Cdoex.dll i Cdosys.dll pomaga w ochronie systemu podlegającego usterce przed próbami jej wykorzystania.
  
    W celu wyłączenia plików Cdoex.dll i Cdosys.dll na serwerach Exchange 2000 Server należy wykonać poniższe czynności.  
    **Uwaga** Czynności należy wykonać w podanej kolejności.
  
    1.  Wykonaj następujące polecenie z wiersza poleceń:  
        **Regsvr32.exe “C:\\Program Files\\Common Files\\Microsoft Shared\\CDO\\cdoex.dll” /u**  
    2.  Wykonaj następujące polecenie z wiersza poleceń:  
        **Regsvr32.exe %windir%\\system32\\cdosys.dll /u**
  
    W celu wyłączenia pliku Cdosys.dll na serwerach z uruchomioną usługą IIS, wpisz następujące polecenie w wierszu poleceń:
  
    1.  **Regsvr32.exe %windir%\\system32\\cdosys.dll /u**
  
    **Wpływ rozwiązania:** Wyrejestrowanie obiektów COM uniemożliwia działanie programów wykorzystujących plik Cdosys.dll lub Cdoex.dll.
  
#### Często zadawane pytania dotyczące usterki obiektów Collaboration Data Objects (CDO) - CAN-2005-1987:
  
**Jaki zakres obejmuje usterka?**  
Jest to usterka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę usterkę, może zdalnie uzyskać pełną kontrolę nad systemem podlegającym usterce. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami.
  
**Co powoduje usterkę?**  
Niesprawdzony bufor w obiektach Collaboration Data Objects (CDO).
  
**Co to są obiekty Collaboration Data Objects (CDO)?**  
Obiekty Collaboration Data Objects (CDO) to składnik modelu Component Object Model (COM) mający na celu między innymi ułatwienie pisania programów tworzących lub modyfikujących internetowe wiadomości pocztowe.
  
**Jakie mogą być skutki wykorzystania usterki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę usterkę, może uzyskać pełną kontrolę nad systemem jej podlegającym.
  
**Kto może wykorzystać usterkę?**  
W serwerze Microsoft Windows lub Microsoft Exchange 2000 usterkę może wykorzystać każdy użytkownik anonimowy, któremu uda się wprowadzić do systemu podlegającego usterce spreparowaną wiadomość.
  
**W jaki sposób osoba atakująca może wykorzystać tę usterkę?**  
Osoba atakująca może spróbować wykorzystać tę usterkę tworząc spreparowaną wiadomość, która uruchomiłaby ujście zdarzeń w systemie podlegającym usterce. Po odpowiednim przetworzeniu mogłoby to umożliwić przejęcie kontroli w systemie, w którym uruchomiony jest protokół SMTP.
  
**Jakie systemy są szczególnie narażone na skutki tej usterki?**  
Zagrożone są przede wszystkim te serwery Microsoft Windows i Microsoft Exchange 2000, w których do przetwarzania wiadomości są wykorzystywane aplikacje wywołujące funkcje bibliotek CDOSYS lub CDOEX.
  
**W jaki sposób działa aktualizacja?**  
Aktualizacja eliminuje usterkę, zmieniając sposób sprawdzania przez obiekt CDO długości wiadomości przed przekazaniem ich do przydzielonego buforu.
  
**Czy usterka była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**
