Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS11-051 - Ważna
======================================================

Luka w zabezpieczeniach składnika rejestrowania w sieci Web Usług certyfikatów w usłudze Active Directory może pozwolić na podniesienie uprawnień (2518295)
-----------------------------------------------------------------------------------------------------------------------------------------------------------

Opublikowano: 14 czerwca 2011 | Zaktualizowano: 15 czerwca 2011

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach składnika rejestrowania w sieci Web Usług certyfikatów w usłudze Active Directory zgłoszoną przez użytkowników. Omawiana luka dotyczy wykorzystania skryptów krzyżowych (XSS) i może pozwolić na podniesienie uprawnień, pozwalając osobie atakującej na wykonanie dowolnych poleceń w witrynie w kontekście zaatakowanego użytkownika. Osoba atakująca, której udałoby się wykorzystać tę lukę w zabezpieczeniach, musiałaby wysłać specjalnie spreparowane łącze i przekonać użytkownika do kliknięcia go. Jednakże w żadnym przypadku osoba atakująca nie może zmusić użytkownika do odwiedzenia witryny sieci Web. Musi natomiast przekonać użytkownika do odwiedzenia tej witryny sieci Web, najczęściej zachęcając do kliknięcia łącza w wiadomości e-mail lub wiadomości błyskawicznej, które przekieruje go do witryny zagrożonej przez lukę.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla wszystkich obsługiwanych wersji systemów Windows Server 2003, Windows Server 2008 oraz Windows Server 2008 R2 z wyjątkiem wersji dla systemów z procesorem Itanium. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach, modyfikując sposób, w jaki witryna rejestracji w sieci Web Usług certyfikatów w usłudze Active Directory sprawdza poprawność parametrów wejściowych. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

**Znane problemy.** W [artykule 2518295 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/2518295) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń. Artykuł ten przedstawia również zalecane obejścia tych problemów.

Luka w zabezpieczeniach Usług certyfikatów w usłudze Active Directory — CVE-2011-1264  
-------------------------------------------------------------------------------------
  
<span></span>
W składniku rejestrowania w sieci Web Usług certyfikatów w usłudze Active Directory występuje luka typu „non-persistent XSS”, która może pozwolić osobie atakującej na umieszczenie skryptu po stronie klienta w instancji przeglądarki Internet Explorer zaatakowanego użytkownika. W witrynie sieci Web skrypt może w imieniu zaatakowanego użytkownika fałszować treść witryny, ujawniać informacje lub wykonywać każdą czynność, którą może wykonać użytkownik.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2011-1264](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1264).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach Usług certyfikatów w usłudze Active Directory — CVE-2011-1264
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   W przypadku użytkowników przeglądarek Internet Explorer 8 i Internet Explorer 9, którzy przechodzą do witryny sieci Web w strefie Internet, funkcja filtru XSS tych przeglądarek uniemożliwia atak. W programach Internet Explorer 8 i 9 funkcja filtru XSS nie jest domyślnie włączona w strefie Intranet.  
-   Osoba atakująca za pośrednictwem sieci Web, która chce wykorzystać tę lukę, może udostępniać witrynę zawierającą odpowiednią stronę sieci Web. Dodatkowo, witryny sieci Web, które akceptują lub udostępniają zawartość wprowadzoną przez użytkownika lub reklamy, jak również witryny niesprawdzone, mogą zawierać specjalnie spreparowane treści, które mogą wykorzystać lukę w zabezpieczeniach. Jednakże w żadnym przypadku osoba atakująca nie może zmusić użytkowników do odwiedzenia takich witryn. Musi natomiast przekonać użytkowników do odwiedzenia tej witryny, najczęściej zachęcając do kliknięcia łącza w wiadomości e-mail lub wiadomości błyskawicznej, które przekieruje użytkowników do witryny osoby atakującej.
  
#### Obejścia problemu spowodowanego przez lukę w zabezpieczeniach Usług certyfikatów w usłudze Active Directory — CVE-2011-1264
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   **Włączenie filtru XSS w programach Internet Explorer 8 i Internet Explorer 9 dla strefy Intranet**
  
    1.  W menu **Narzędzia** programu Internet Explorer kliknij polecenie **Opcje internetowe**.  
    2.  W oknie dialogowym **Opcje internetowe** kliknij kartę **Zabezpieczenia**, a następnie ikonę **Lokalny intranet**.  
    3.  W sekcji **Poziom zabezpieczeń dla tej strefy** kliknij przycisk **Poziom niestandardowy.**  
    4.  Przewiń w dół do opcji **Włącz filtr XSS** (przy końcu listy)  
    5.  Zmień domyślną wartość **Wyłącz** na **Włącz**  
    6.  Kliknij przycisk **OK**, a następnie **Tak**, gdy zostanie wyświetlone okno dialogowe **Czy na pewno chcesz zmienić ustawienia dla tej strefy?**.  
    7.  Kliknij przycisk **OK** w oknie dialogowym **Opcje internetowe**.
  
    **Wpływ obejścia**. Witryny wewnętrzne, które wcześniej nie były oznaczone jako zagrożone luką XSS, zostaną oznaczone.
  
    **Cofnięcie obejścia.**
  
    1.  W menu **Narzędzia** programu Internet Explorer kliknij polecenie **Opcje internetowe**.  
    2.  W oknie dialogowym **Opcje internetowe** kliknij kartę **Zabezpieczenia**, a następnie ikonę **Lokalny intranet**.  
    3.  W sekcji **Poziom zabezpieczeń dla tej strefy** kliknij przycisk **Poziom niestandardowy.**  
    4.  Przewiń w dół do opcji **Włącz filtr XSS** (przy końcu listy)  
    5.  Zmień domyślne ustawienie **Włącz** na **Wyłącz**  
    6.  Kliknij przycisk **OK**, a następnie **Tak**, gdy zostanie wyświetlone okno dialogowe **Czy na pewno chcesz zmienić ustawienia dla tej strefy?**.  
    7.  Kliknij przycisk **OK** w oknie dialogowym **Opcje internetowe**.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach Usług certyfikatów w usłudze Active Directory — CVE-2011-1264
  
**Jaki zakres obejmuje luka?**    
Jest to luka typu „non-persistent XSS”, która może pozwolić na podniesienie uprawnień.
  
**Skąd bierze się luka?**    
Przyczyną tej luki jest nieprawidłowy sposób sprawdzania poprawności parametrów żądania w witrynie rejestracji w sieci Web Usług certyfikatów w usłudze Active Directory.
  
**Co to są skrypty krzyżowe (XSS)?**    
Skrypty krzyżowe (XSS) wiążą się z luką w zabezpieczeniach umożliwiającą osobie atakującej „wstrzyknięcie” kodu skryptu do odpowiedzi na żądanie witryny sieci Web. Skrypt ten jest następnie uruchamiany przez aplikację wysyłającą żądanie, zazwyczaj przeglądarkę internetową. W witrynie sieci Web, której dotyczy luka, skrypt może następnie w imieniu zaatakowanego użytkownika fałszować treść witryny, ujawniać informacje lub wykonywać każdą czynność, którą może wykonać użytkownik.
  
**Jaki składnik jest dotknięty przez omawianą lukę?**    
Składnikiem zagrożonym przez omawianą lukę jest Active Directory Certificate Services Web Enrollment.
  
**Jakie mogą być skutki wykorzystania luki przez osobę atakującą?**     
Osoba atakująca, która pomyślnie wykorzystała tę lukę, może „wstrzyknąć” skrypt po stronie klienta, korzystając z instancji przeglądarki Internet Explorer. W witrynie sieci Web skrypt może w imieniu zaatakowanego użytkownika fałszować treść witryny, ujawniać informacje lub wykonywać każdą czynność, którą może wykonać użytkownik.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**     
Osoba atakująca za pośrednictwem poczty e-mail może wykorzystać tę lukę, wysyłając do użytkownika specjalnie spreparowane łącze i nakłaniając go do kliknięcia tego łącza.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**     
Szczególnie zagrożone są serwery, na których działa rola Active Directory Certificate Services Web Enrollment. Serwery pozbawione tej roli nie są zagrożone.
  
**W jaki sposób działa aktualizacja?**     
Aktualizacja usuwa lukę w zabezpieczeniach, modyfikując sposób, w jaki witryna rejestracji w sieci Web Usług certyfikatów w usłudze Active Directory sprawdza poprawność parametrów wejściowych.
  
**Czy luka była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od współpracujących źródeł.
  
**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.
