Biuletyn zabezpieczeń firmy Microsoft MS07-022 - Ważna
======================================================

Luka w zabezpieczeniach jądra systemu Windows może pozwolić na podniesienie poziomu uprawnień (931784)
------------------------------------------------------------------------------------------------------

Opublikowano: 10 kwietnia 2007 | Zaktualizowano: 26 czerwca 2007

**Wersja:** 2.0

#### Streszczenie

**Kto powinien przeczytać ten dokument:** Klienci korzystający z systemu Microsoft Windows

**Wpływ luki:** Podniesienie uprawnień

**Maksymalny wskaźnik ważności:** Ważne

**Zalecenie:** Klienci powinni jak najszybciej zastosować tę aktualizację

**Zastąpienie aktualizacji zabezpieczeń:** Ten biuletyn zastępuje poprzednią aktualizację zabezpieczeń. Szczegółowe informacje na ten temat znajdują się w sekcji „Często zadawane pytania" w niniejszym biuletynie.

**Lokalizacje, z których można pobrać przetestowane oprogramowanie i aktualizacje zabezpieczeń:**

**Programy, których dotyczy problem:**

-   Microsoft Windows 2000 z dodatkiem Service Pack 4 — [Pobierz aktualizację](http://www.microsoft.com/downloads/details.aspx?familyid=b3599afb-7673-4ef6-a2b1-d77e39fd782c&displaylang=pl)
-   Microsoft Windows XP z dodatkiem Service Pack 2 - [Pobierz aktualizację](http://www.microsoft.com/downloads/details.aspx?familyid=eeaee4a7-4858-4b6b-9c6d-a9f1eae19b51&displaylang=pl)
-   Microsoft Windows Server 2003, Microsoft Windows Server 2003 z dodatkiem Service Pack 1 i Microsoft 2003 z dodatkiem Service Pack 2 — [Pobierz aktualizację](http://www.microsoft.com/downloads/details.aspx?familyid=5e94a29c-7ec0-4459-92eb-d43b524fd6fd&displaylang=pl)

**Programy, których nie dotyczy problem:**

-   Microsoft Windows XP Professional x64 Edition oraz Microsoft Windows XP Professional x64 Edition z dodatkiem Service Pack 2
-   Microsoft Windows Server 2003 x64 Edition oraz Microsoft Windows Server 2003 x64 Edition z dodatkiem Service Pack 2

Często zadawane pytania dotyczące tej aktualizacji zabezpieczeń
---------------------------------------------------------------


**Dlaczego firma Microsoft wydała ten biuletyn ponownie dnia 26 czerwca 2007 r.?**  
Firma Microsoft zaktualizowała ten biuletyn, by poinformować klientów korzystających z systemu Windows 2000 z dodatkiem Service Pack 4 zainstalowanego na komputerach NEC 98 o ukazaniu się zmodyfikowanej wersji aktualizacji zabezpieczeń. W systemach tego rodzaju może wystąpić problem podczas próby połączenia się z usługą Windows Update, spowodowany niewłaściwą kompilacją aktualizacji zabezpieczeń dla komputerów NEC 98. Pierwotna wersja aktualizacji zabezpieczeń skutecznie rozwiązywała problemy opisane w niniejszym biuletynie.

Firma Microsoft zaleca, by klienci korzystający z komputerów NEC 98 z systemem Windows 2000 z dodatkiem Service Pack 4 pobrali zmodyfikowaną wersję aktualizacji z Centrum pobierania Microsoft, by upewnić się, że usługa Windows Update będzie udostępniać przyszłe aktualizacje w sposób prawidłowy. Użytkownicy komputerów NEC 98 z systemem Windows 2000 z dodatkiem Service Pack 4 mogą także odinstalować tę aktualizację zabezpieczeń, co pozwoli na automatyczne jej pobranie przez usługę Windows Update.

Dodatkowe informacje na ten temat można znaleźć w [artykule 931784 w bazie wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/931784/). W [artykule 931784 w bazie wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/931784/) opisano znane obecnie problemy, jakie mogą napotkać klienci po zainstalowaniu zmodyfikowanej aktualizacji zabezpieczeń.

Szczegółowe informacje o luce w zabezpieczeniach
------------------------------------------------


#### Luka w zabezpieczeniach jądra umożliwiająca lokalne podniesienie uprawnień — CVE-2007-1206:

W jądrze systemu Windows luka umożliwiająca podniesienie poziomu uprawnień związana z niepoprawnymi uprawnieniami mapowanego segmentu pamięci. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.

#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach jądra umożliwiającej lokalne podniesienie poziomu uprawnień — CVE-2007-1206:

Osoba atakująca musi dysponować prawidłowymi poświadczeniami logowania oraz być w stanie zalogować się lokalnie, aby wykorzystać możliwości stwarzane przez lukę. Nie jest możliwe wykorzystanie luki w sposób zdalny lub przez użytkowników anonimowych.

#### Obejścia problemu spowodowanego przez lukę w zabezpieczeniach jądra umożliwiającą lokalne podniesienie poziomu uprawnień — CVE-2007-1206:

Nie znaleziono żadnego obejścia tej luki.

#### Często zadawane pytania dotyczące luki w zabezpieczeniach jądra umożliwiającej lokalne podniesienie poziomu uprawnień — CVE-2007-1206:

**Dlaczego niniejszy biuletyn jest udostępniany ponownie?**  
Klienci korzystający z systemu Windows 2000 na komputerach NEC 98 mogli nie otrzymać aktualizacji zabezpieczeń 931784 w sposób prawidłowy. Firma Microsoft wydała biuletyn ponownie, aby rozwiązać ten problem. Ponowne wydanie biuletynu dotyczy tylko i wyłącznie klientów korzystających z komputerów NEC 98.

**Jaki zakres obejmuje luka?**  
W jądrze systemu Windows luka umożliwiająca podniesienie poziomu uprawnień związana z niepoprawnymi uprawnieniami mapowanego segmentu pamięci. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Aby podjąć próbę wykorzystania luki, osoba atakująca musi zalogować się lokalnie do systemu.

**Co powoduje lukę?**  
Podczas mapowania segmentu pamięci jądro systemu Windows zezwala na użycie niepoprawnych uprawnień.

**Jakie mogą być skutki wykorzystania luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może uruchomić dowolny kod w systemie użytkownika z podniesionymi uprawnieniami. W ten sposób osoba atakująca może uzyskać pełną kontrolę nad danym systemem.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Aby wykorzystać lukę, osoba atakująca musi najpierw zalogować się do systemu. Następnie musi ona uruchomić specjalny program, umożliwiający wykorzystanie luki, co pozwala na uzyskanie pełnej kontroli nad systemem, którego luka dotyczy.

**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Szczególnie narażone są stacje robocze i serwery terminalowe. Zagrożenie serwerów może być większe, jeśli administratorzy zezwolą użytkownikom na logowanie do tych serwerów i uruchamianie programów. Jednak zestawy najlepszych praktyk zdecydowanie to odradzają.

**Czy lukę można wykorzystać zdalnie za pomocą Internetu?**  
Nie. Osoba atakująca musi być zalogowana do określonego systemu, jaki ma zamiar zaatakować. Intruz nie ma możliwości zdalnego pobrania i uruchomienia programu, wykorzystując tę lukę.

**W jaki sposób działa aktualizacja?**  
Aktualizacja usuwa lukę, modyfikując sposób, w jaki jądro systemu Windows sprawdza poprawność uprawnień podczas mapowania pamięci.

**Czy luka była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**  
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.

**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki w czasie, gdy opublikowano niniejszy biuletyn?**  
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.
