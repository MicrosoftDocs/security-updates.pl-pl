Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS07-061 - Krytyczna
==========================================================

Luka w zabezpieczeniach mechanizmu obsługi identyfikatorów URI w systemie Windows potencjalnie umożliwia zdalne wykonanie kodu (943460)
---------------------------------------------------------------------------------------------------------------------------------------

Opublikowano: 13 listopada 2007 | Zaktualizowano: 16 stycznia 2008

**Wersja:** 1.2

### Informacje ogólne

#### Streszczenie

Ta aktualizacja ma na celu usunięcie luki w zabezpieczeniach zgłoszonej przez organizację publiczną. W sposobie, w jaki powłoka Windows obsługuje przekazywane do niej specjalnie spreparowane identyfikatory URI, występuje luka. Gdyby powłoka systemu Windows niewystarczająco sprawdzała poprawność identyfikatorów URI, osoba atakująca mogłaby wykorzystać lukę w zabezpieczeniach i uruchomić dowolny kod. Firma Microsoft znalazła sposoby na wykorzystanie tej luki tylko w systemach korzystających z programu Internet Explorer 7. Luka ta istnieje jednak w pliku systemu Windows, shell32.dll, który znajduje się we wszystkich obsługiwanych wersjach systemów Windows XP i Windows Server 2003.

Niniejsza aktualizacja zabezpieczeń ma krytyczny wskaźnik ważności dla wszystkich obsługiwanych wersji systemu Windows XP i Windows Server 2003. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Niniejsza aktualizacja zabezpieczeń eliminuje lukę, zmieniając sposób, w jaki powłoka systemu Windows obsługuje nieprawidłowe identyfikatory URI. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

Niniejsza aktualizacja zabezpieczeń dotyczy również luki w zabezpieczeniach opisanej po raz pierwszy w [Poradniku zabezpieczeń firmy Microsoft 943521 (j.ang.)](http://technet.microsoft.com/security/advisory/943521). Więcej informacji można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej określonej luce w następnej sekcji, Informacje o luce w zabezpieczeniach.

**Zalecenie.** Firma Microsoft zaleca klientom natychmiastowe zastosowanie tej aktualizacji.

**Znane problemy.** W [artykule 9434601 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/943460) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń. Artykuł ten przedstawia również zalecane obejścia tych problemów.

Luka w obsłudze identyfikatorów URI w systemie Windows - CVE-2007-3896
----------------------------------------------------------------------

<span></span>
W sposobie, w jaki powłoka Windows obsługuje przekazywane do niej specjalnie spreparowane identyfikatory URI, występuje luka. Osoba atakująca może wykorzystać tę lukę, umieszczając specjalnie spreparowany identyfikator URI w aplikacji lub załączniku, co może spowodować zdalne wykonanie kodu.

Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2007-3896 (j.ang.)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896).

#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach sposobu, w jaki system Windows obsługuje identyfikatory URI - CVE-2007-3896

Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:

-   System Windows 2000 z dodatkiem Service Pack 4 nie podlega omawianej luce
-   System Windows Vista nie podlega omawianej luce
-   System Windows Vista x64 Edition nie podlega omawianej luce
-   Firma Microsoft nie znalazła żadnych sposobów na wykorzystanie tej luki w żadnym systemie operacyjnym Windows, korzystającym z przeglądarki Microsoft Internet Explorer w wersji 6
-   Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać takie same uprawnienia, jak użytkownik lokalny. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.
-   W przypadku ataków wykorzystujących lukę za pomocą poczty elektronicznej użytkownicy czytający wiadomości e-mail w postaci zwykłego tekstu są w mniejszym stopniu zagrożeni tą luką. Aby powstało większe zagrożenie ze strony omawianej luki, użytkownicy musieliby kliknąć łącze do złośliwej witryny sieci Web lub otworzyć załącznik.

#### Obejścia problemu wynikającego z luki w zabezpieczeniach sposobu, w jaki system Windows obsługuje identyfikatory URI - CVE-2007-3896

Firma Microsoft nie znalazła żadnego obejścia tej luki.

#### Często zadawane pytania dotyczące luki w zabezpieczeniach sposobu, w jaki system Windows obsługuje identyfikatory URI - CVE-2007-3896

**Jaki zakres obejmuje ta luka?**  
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać takie same uprawnienia, jak zalogowany użytkownik.

Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca, która pomyślnie wykorzysta tę lukę, może uzyskać pełną kontrolę nad systemem, którego ta luka dotyczy. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.

**W jaki sposób powstaje luka?**  
Powłoka systemu Windows w niewystarczający sposób obsługuje nieprawidłowe identyfikatory URI.

**Co to jest identyfikator URI?**
Uniform Resource Identifier (URI) to ciąg znaków służący do oddziaływania na zasoby w Internecie lub sieci oraz do ich identyfikacji. Adres URL to typowy przykład identyfikatora URI, który odnosi się do zasobu, jakim jest witryna sieci Web. Więcej informacji na temat identyfikatorów URI można znaleźć w dokumencie [RFC 2396 (j.ang.)](http://www.ietf.org/rfc/rfc2396.txt).

**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się przekonać użytkownika do otwarcia załącznika wiadomości e-mail lub kliknięcia łącza prowadzącego do witryny sieci Web spreparowanej przez osobę atakującą, może wykonać dowolny kod jako zalogowany użytkownik. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Osoba atakująca musiałaby stworzyć specjalnie spreparowany identyfikator URI, a następnie przekazać go jako dane wejściowe dla aplikacji działającej w systemie zawierającym lukę, która następnie próbowałaby uzyskać dostęp do zasobu określanego przez ten identyfikator. Aplikacje, które przyjmują identyfikatory URI jako dane wejściowe ze źródeł niezaufanych, takich jak załączniki wiadomości e-mail, dokumenty lub dane z sieci, zakładając, że są one bezpieczne, są narażone na wykorzystanie tej luki. W określonych warunkach przetworzenie specjalnie spreparowane dane wejściowe identyfikatora URI może spowodować wykonanie dowolnego kodu. Aby wykorzystać tę lukę, osoba atakująca musi przekonać użytkownika do otwarcia załącznika, uruchomienia aplikacji lub odwiedzenia witryny sieci Web. Osoba atakująca może udostępniać specjalnie spreparowaną witrynę sieci Web, której celem jest wykorzystanie tej luki w zabezpieczeniach w programie Internet Explorer, a następnie przekonać użytkownika do wyświetlenia takiej witryny. Omawiana luka w zabezpieczeniach może także być wykorzystana przez witryny sieci Web, do których się włamano lub witryny, które akceptują lub udostępniają treści lub reklamy wprowadzone przez użytkownika. Witryny te mogą zawierać specjalnie spreparowane identyfikatory URI, które mogą wykorzystywać omawianą lukę w zabezpieczeniach. Może również istnieć możliwość wyświetlenia specjalnie spreparowanych identyfikatorów URI z wykorzystaniem reklam banerowych lub innych metod jej dostarczenia do systemów, których dotyczy luka.

**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Luce podlegają komputery działające pod kontrolą obsługiwanych wersji systemu Windows XP i Windows Server 2003 z zainstalowaną przeglądarką Internet Explorer 7.

**W jaki sposób działa aktualizacja? **
Aktualizacja eliminuje lukę, zmieniając sposób, w jaki powłoka systemu Windows obsługuje nieprawidłowe identyfikatory URI.

**Na komputerze nie jest zainstalowany program Windows Internet Explorer w wersji 7. Dlaczego otrzymuję tę aktualizację? **
Luka istnieje w pliku systemu Windows, shell32.dll, który znajduje się we wszystkich obsługiwanych wersjach systemów Windows XP i Windows Server 2003. Firma Microsoft nie znalazła żadnych sposobów na wykorzystanie tej luki w systemach z programem Internet Explorer 6, który jest przeglądarką sieci Web dołączaną do tych systemów operacyjnych. Niniejsza aktualizacja zabezpieczeń jest udostępniana jako dodatkowe wzmocnienie ochrony komputera wszystkim użytkownikom obsługiwanych wersji systemów Windows XP i Windows Server 2003, bez względu na zainstalowaną wersję programu Internet Explorer.

**Używam systemu Windows** **Windows, czy omawiana luka stanowi dla mnie zagrożenie? **
Nie. System Windows Vista nie podlega omawianej luce. Program Windows Internet Explorer 7 jest dostarczany z systemem Windows Vista, ale omawiana luka w zabezpieczeniach nie dotyczy powłoki Windows w systemie Windows Vista.

**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn? **
Tak. W chwili pierwszego wydania tego biuletynu zabezpieczeń luka ta była powszechnie znana. Została ona zarejestrowana w bazie danych znanych luk i zagrożeń pod numerem [CVE-2007-3896 (j.ang.)](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896). Po raz pierwszy opisano ją w [Poradniku zabezpieczeń firmy Microsoft 943521](http://technet.microsoft.com/security/advisory/943521).

**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki przed publikacją niniejszego biuletynu? **
Tak. Przed opublikowaniem biuletynu zabezpieczeń firma Microsoft otrzymała informację o wykorzystywaniu tej luki.

**Czy zastosowanie aktualizacji pomaga w ochronie systemów klientów przed publicznie dostępnym kodem usiłującym wykorzystać tę lukę? **
Tak. Ta aktualizacja zabezpieczeń eliminuje aktualnie wykorzystywaną lukę. Informacje o usuniętej luce w zabezpieczeniach zostały zarejestrowane w bazie danych znanych luk i zagrożeń pod numerem [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896).

    Aby sprawdzić, czy ta aktualizacja zabezpieczeń została zainstalowana w systemie, którego dotyczy luka, można skorzystać z narzędzia Microsoft Baseline Security Analyzer (MBSA). -   **Weryfikacja wersji pliku**

    Ze względu na to, że dostępne są różne wersje systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.
        **Uwaga** W niektórych wersjach zainstalowanych systemów operacyjnych lub programów niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego na komputerze, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.
