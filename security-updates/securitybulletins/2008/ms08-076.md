Biuletyn zabezpieczeń firmy Microsoft MS08-076 - Ważna
======================================================

Luki w zabezpieczeniach programu Windows Media Components umożliwiają zdalne wykonanie kodu (959807)
----------------------------------------------------------------------------------------------------

Opublikowano: 9 grudnia 2008 | Zaktualizowano: 24 listopada 2009

**Wersja:** 5.0

### Informacje ogólne

#### Streszczenie

Ta aktualizacja zabezpieczeń usuwa dwie zgłoszone przez użytkowników luki w zabezpieczeniach następujących składników Windows Media: Windows Media Player, Windows Media Format Runtime i Windows Media Services. Najpoważniejsza z luk umożliwia zdalne wykonanie kodu. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca, która pomyślnie wykorzysta tę lukę, może uzyskać pełną kontrolę nad systemem, którego ta luka dotyczy. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla programów Windows Media Player 6.4, Windows Media Format Runtime 7.1, Windows Media Format Runtime 9.0, Windows Media Format Runtime 9.5, Windows Media Format Runtime 11, Windows Media Services 4.1, Windows Media Services 9 Series i Windows Media Services 2008. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa pierwszą lukę, zmieniając sposób sprawdzana poprawności odpowiedzi uwierzytelniania Windows Media. Aktualizacja zabezpieczeń usuwa drugą lukę, zapewniając traktowanie serwerów używających adresów ISATAP jako zewnętrznych przez klientów pakietu Windows Media. Więcej informacji na temat luk może znaleźć w podsekcjach „Często zadawane pytania” dla pozycji określonych luk w zabezpieczeniach, znajdujących się poniżej następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Firma Microsoft zaleca klientom jak najszybsze zastosowanie tej aktualizacji.

**Znane problemy.** Brak

#### Programy, których dotyczy problem i programy, których nie dotyczy problem

Oprogramowanie wymienione poniżej zostało przetestowane w celu ustalenia, których wersji dotyczy problem. Dla pozostałych wersji dobiegł końca okres pomocy technicznej bądź luka w zabezpieczeniach ich nie dotyczy. Aby zapoznać się z zasadami cyklu pomocy technicznej dotyczącymi używanej wersji oprogramowania, odwiedź [witrynę zasad cyklu pomocy technicznej firmy Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Często zadawane pytania dotyczące tej aktualizacji zabezpieczeń
---------------------------------------------------------------

**Dlaczego 24 listopada 2009 r. ponownie wydano niniejszy biuletyn?**  
Firma Microsoft opublikowała ponownie niniejszy biuletyn, aby zaoferować nowy pakiet aktualizacji przeznaczonych specjalnie dla systemu Windows XP Embedded. Miało to na celu usunięcie problemów z wykrywaniem i instalacją oryginalnych pakietów aktualizacji dla systemu Windows XP, które znajdują się w tym biuletynie.

Ponadto firma Microsoft wprowadziła zmiany do istniejących pakietów aktualizacji dla systemów Windows XP z dodatkiem Service Pack 2 (KB952069) i Windows XP z dodatkiem Service Pack 3 (KB952069), aby zapobiec ich oferowaniu dla systemów Windows XP Embedded. Klienci, którzy zainstalowali już tę aktualizację, uzyskali ochronę i nie muszą podejmować żadnych czynności.

Klienci korzystający z systemu Windows XP Embedded powinni zainstalować tę aktualizację przy najbliższej okazji. Można również ręcznie pobrać aktualizację dla systemu [Windows XP Embedded z dodatkiem Service Pack 2](http://www.microsoft.com/downloads/details.aspx?familyid=c8f59b79-7d09-4976-a678-9aef5975c186) (KB972187) oraz dla systemu [Windows XP Embedded z dodatkiem Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=502dcf76-aec3-48bf-930d-0c02b1604cd1) (KB972187). Niniejsza ponowna publikacja nie dotyczy innych aktualizacji.

**Co to jest system operacyjny Windows XP Embedded?**  
System operacyjny Microsoft Windows XP Embedded to wersja systemu Windows XP Professional w formie składnika, która zapewnia dostępność szerokiego wachlarza funkcji tego systemu dla urządzeń osadzonych. System Windows XP Embedded został zoptymalizowany pod kątem zastosowań w branży handlu detalicznego i oferuje wbudowane funkcje pracy w sieci oraz opartą na standardach obsługę technologii plug-and-play. System Windows XP Embedded umożliwia łączenie szeregu urządzeń peryferyjnych przeznaczonych dla branży detalicznej oraz hotelowo-gastronomicznej w pojedynczy system usługowy POS. Aby uzyskać więcej informacji na temat systemów operacyjnych Windows XP Embedded, przejdź do strony [Microsoft Windows Embedded Developer Center](http://msdn.microsoft.com/en-us/windowsembedded/default.aspx).

**Dlaczego 13 stycznia 2009 r. ponownie wydano niniejszy biuletyn?**  
Firma Microsoft opublikowała ponownie niniejszy biuletyn, aby zaoferować nowe pakiety aktualizacyjne dla programu Windows Media Format Runtime 9.5 zainstalowanego w systemie Windows XP z dodatkiem Service Pack 2 (KB952069) i w systemie Windows XP z dodatkiem Service Pack 3 (KB952069). Nowe pakiety aktualizacji zawierają rozwiązanie problemu dotyczącego instalacji, którego skutkiem jest niekompletna instalacja w systemach Windows XP korzystających z programu Windows Media Format Runtime 9.5 w sytuacji, kiedy klienci zainstalowali osobną aktualizację KB944110, zanim przystąpili do instalacji aktualizacji KB952069. Tacy klienci są nadal zagrożeni tym problemem i powinni pobrać oraz zainstalować ponownie wydane pakiety aktualizacji. Ponowna publikacja nie jest przeznaczona dla innych aktualizacji. Klienci, którzy korzystają z wszystkich pozostałych obsługiwanych i dotkniętych problemem wersji programu Windows Media Player, Windows Media Format Runtime oraz Windows Media Services, którzy zainstalowali już oryginalne aktualizacje zabezpieczeń, nie muszą podejmować dalszych działań. Luka w zabezpieczeniach związana z główną nazwą usługi (SPN) — CVE-2008-3009
----------------------------------------------------------------------------

W składnikach pakietu Windows Media występuje luka związana z odbiciem poświadczeń, która umożliwia osobie atakującej wykonanie kodu z uprawnieniami użytkownika lokalnego lub poświadczeniami dystrybucji programu Windows Media Services. Przyczyną występowania tej luki jest niedoskonałość implementacji głównej nazwy usługi (SPN, Service Principal Name) w składnikach Windows Media.

Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2008-3009](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3009).

#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach związanej z główną nazwą usługi (SPN) — CVE-2008-3009

Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:

-   Program Windows Media Services jest składnikiem opcjonalnym na wszystkich platformach Windows Server i nie jest instalowany domyślnie.
-   Osoba atakująca za pośrednictwem sieci Web, która chce wykorzystać te luki, musi udostępniać witrynę zawierającą odpowiednią stronę sieci Web. Intruz nie może w żaden sposób zmusić użytkowników do odwiedzenia szkodliwej witryny. Musi natomiast przekonać ich do jej odwiedzenia, zazwyczaj poprzez kliknięcie łącza do tej witryny. Po kliknięciu łącza użytkownik jest proszony o wykonanie kilku działań. Atak może zostać przeprowadzony tylko po ich wykonaniu.
-   Program Windows Media Format Runtime 11 jest dostępny w systemach Windows Server 2008 i Windows Server 2008 dla systemów z procesorem x64 tylko po zainstalowaniu usługi Środowisko pulpitu.

#### Obejścia dotyczące luki w zabezpieczeniach związanej z główną nazwą usługi (SPN) — CVE-2008-3009

Firma Microsoft nie znalazła żadnego obejścia tej luki.

#### Często zadawane pytania dotyczące luki w zabezpieczeniach związanej z główną nazwą usługi (SPN) — CVE-2008-3009

**Jaki zakres obejmuje ta luka?**  
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uruchomić kod z prawami takiego samego poziomu, jak prawa zalogowanego użytkownika. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca, która pomyślnie wykorzysta tę lukę, może uzyskać pełną kontrolę nad systemem, którego ta luka dotyczy. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.  Osoba atakująca, której uda się wykorzystać tę lukę, może także wykonać kod z poświadczeniami dystrybucji programu Windows Media Services. W takim przypadku osoba atakująca mogłaby wykonać kod z prawami konta usługi sieciowej.

**W jaki sposób powstaje luka?**  
Składniki Windows Media nieprawidłowo wybierają funkcje ochrony przed odbiciem poświadczeń NTMLM, które zabezpieczają przed odbiciem poświadczeń użytkownika z powrotem do jego systemu i wykorzystaniem ich do podszycia się pod użytkownika.

**Co to są nazwy główne usługi (Service Principle Names , SPN)?**  
Nazwa główna usługi (SPN) to niepowtarzalna nazwa umożliwiająca klientowi identyfikację wystąpienia usługi. SPN informuje o typie tożsamości usługi używanym do jej uwierzytelniania wobec klienta. Klient, który chce połączyć się z usługą, lokalizuje jej wystąpienie, układa dla niej nazwę SPN, a następnie łączy się z usługą i przedstawia jej nazwę SPN w celu uwierzytelnienia. Dodatkowe informacje na temat głównej nazwy usługi znajdują się w artykule [Service Identity and Authentication](http://msdn.microsoft.com/en-us/library/ms733130.aspx) (Tożsamość i uwierzytelnianie usług). Dodatkowe informacje dotyczące prawidłowego formatowania nazw SPN znajdują się w artykule [Name Formats for Unique SPNs](http://msdn.microsoft.com/en-us/library/ms677601(vs.85).aspx) (Formaty unikatowych nazw SPN).

**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać takie same uprawnienia, jak zalogowany użytkownik. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca, która pomyślnie wykorzysta tę lukę, może uzyskać pełną kontrolę nad systemem, którego ta luka dotyczy. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Ta luka w zabezpieczeniach wymaga, aby użytkownik składników pakietu Windows Media, których dotyczy luka, uzyskał dostęp do niebezpiecznego serwera. Osoba atakująca może udostępnić niebezpieczny udział na serwerze lub witrynę sieci Web wymagającą uwierzytelnienia. Jeśli klient połączy się z niebezpiecznym serwerem i zostanie uwierzytelniony, osoba atakująca może wykorzystać poświadczenia klienta, odbijając je w jego stronę lub przesyłając na inne serwery, udając klienta. Intruz nie może w żaden sposób zmusić użytkowników do odwiedzenia specjalnie spreparowanego udziału lub witryny. Musi natomiast przekonać ich do odwiedzenia tego udziału lub witryny sieci Web, najczęściej zachęcając do kliknięcia łącza w wiadomości e-mail lub wiadomości błyskawicznej, które przekieruje użytkowników do witryny osoby atakującej.

**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Narażony jest każdy system, w którym wykorzystywana jest objęta luką wersja programu Windows Media Player, Windows Media Format Runtime lub Windows Media Services. Najbardziej narażone na skutki wykorzystania omawianej luki są systemy często używane przez te składniki, takie jak stacje robocze i serwery terminalowe.

Luka w zabezpieczeniach protokołu ISATAP — CVE-2008-3010
--------------------------------------------------------

W obsługiwanych wersjach składników Windows Media istnieje luka umożliwiająca ujawnienie informacji, które może spowodować ujawnienie poświadczeń NTLM. Każdy składnik Windows Media uzyskujący dostęp do adresu URL przy użyciu protokołu ISATAP może spowodować wyciek poświadczeń NTLM użytkownika do serwera obsługującego dany URL. Dzięki temu osoba atakująca z zewnątrz strefy intranetowej może zebrać poświadczenia NTLM dotyczące środowiska przedsiębiorstwa.

Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2008-3010](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3010).

#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach protokołu ISATAP — CVE-2008-3010

Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:

-   Sprawdzone metody działania stosowane w zaporach oraz standardowe domyślne konfiguracje zapór mogą pomóc w zabezpieczeniu zasobów sieciowych przed atakami spoza obszaru działania przedsiębiorstwa. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów i protokołów.
-   Osoba atakująca za pośrednictwem sieci Web, która chce wykorzystać te luki, musi udostępniać witrynę zawierającą odpowiednią stronę sieci Web. Intruz nie może w żaden sposób zmusić użytkowników do odwiedzenia szkodliwej witryny. Musi natomiast przekonać ich do jej odwiedzenia, zazwyczaj poprzez kliknięcie łącza do tej witryny. Po kliknięciu łącza użytkownik jest proszony o wykonanie kilku działań. Atak może zostać przeprowadzony tylko po ich wykonaniu.
-   Program Windows Media Format Runtime 11 jest dostępny w systemach Windows Server 2008 i Windows Server 2008 dla systemów z procesorem x64 tylko po zainstalowaniu usługi Środowisko pulpitu.

#### Obejścia dotyczące luki w zabezpieczeniach protokołu ISATAP — CVE-2008-3010

Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:

-   **Zablokowanie protokołu IP Typ 41 (ISATAP) na zaporze**  

    Protokół ISATAP został zdefiniowany jako typ protokołu 41 w specyfikacji [RFC4214: Intra-Site Automatic Tunnel Addressing Protocol (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt). Zablokowanie tego typu protokołu na zaporze może pomóc w zabezpieczeniu systemów chronionych zaporą przed próbami wykorzystania opisywanej luki. W celu zapobieżenia atakom mogącym wykorzystywać inne porty lub protokoły firma Microsoft zaleca blokowanie całego niepożądanego ruchu przychodzącego z Internetu.

#### Często zadawane pytania dotyczące luki w zabezpieczeniach protokołu ISATAP — CVE-2008-3010

**Jaki zakres obejmuje ta luka?**  
Ta luka może doprowadzić do ujawnienia informacji. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać poświadczenia NTLM zalogowanego użytkownika. Poświadczenia te mogą następnie zostać odbite z powrotem do systemu użytkownika w celu wykonania kodu z takimi samymi prawami, jak prawa zalogowanego użytkownika.

**W jaki sposób powstaje luka?**  
Podczas łączenia się z serwerem o adresie ISATAP składniki Windows Media, których dotyczy luka, błędnie traktują zasoby Internetu jako część strefy Lokalny intranet.

**Co to jest ISATAP?**  
Protokół ISATAP (Intra-Site Automatic Tunnel Addressing Protocol) zapewnia łączność przy użyciu protokołu IPv6 w intranecie z protokołem IPv4. Specyfikacja RFC dotycząca protokołu ISATAP została udostępniona przez grupę IETF (Internet Engineering Task Force). Informacje na temat protokołu ISATAP znajdują się w specyfikacji [RFC4214: Intra-Site Automatic Tunnel Addressing Protocol (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt).

**Co to jest NTLM?**  
NTLM to protokół uwierzytelniania bazujący na mechanizmie wyzwanie/odpowiedź.

**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może zdobyć poświadczenia NTLM użytkownika.  Poświadczenia te mogą następnie zostać odbite z powrotem do systemu użytkownika w celu wykonania kodu z takimi samymi prawami, jak prawa zalogowanego użytkownika.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Ta luka w zabezpieczeniach wymaga, aby użytkownik składników pakietu Windows Media, których dotyczy luka, uzyskał dostęp do niebezpiecznego serwera. Osoba atakująca może udostępnić niebezpieczny udział na serwerze lub witrynę sieci Web wymagającą uwierzytelnienia. Jeśli klient połączy się z niebezpiecznym serwerem i zostanie uwierzytelniony, osoba atakująca może wykorzystać poświadczenia klienta, odbijając je w jego stronę lub przesyłając na inne serwery, udając klienta. Intruz nie może w żaden sposób zmusić użytkowników do odwiedzenia specjalnie spreparowanego udziału lub witryny. Musi natomiast przekonać ich do odwiedzenia tego udziału lub witryny sieci Web, najczęściej zachęcając do kliknięcia łącza w wiadomości e-mail lub wiadomości błyskawicznej, które przekieruje użytkowników do witryny osoby atakującej.

**Jakie systemy są szczególnie narażone na skutki tej luki?**

Najbardziej narażone na skutki wykorzystania omawianej luki są systemy często używane przez te składniki, takie jak stacje robocze i serwery terminalowe.

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.  
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.

        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.
