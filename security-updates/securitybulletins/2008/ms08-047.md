Biuletyn zabezpieczeń firmy Microsoft MS08-047 - Ważna
======================================================

Luka w zabezpieczeniach przetwarzania zasad protokołu IPsec umożliwia ujawnienie informacji (953733)
----------------------------------------------------------------------------------------------------

Opublikowano: 12 sierpnia 2008 | Zaktualizowano: 13 sierpnia 2008

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa zgłoszoną przez użytkowników lukę w sposobie stosowania pewnych reguł protokołu IPSec systemu Windows. Ta luka może spowodować, że systemy będą ignorować zasady protokołu IPSec i przesyłać dane w sieci w postaci czystego tekstu. To z kolei umożliwi ujawnienie informacji, które powinny być zaszyfrowane podczas przesyłania w sieci. Osoba atakująca monitorująca ruch sieciowy może wyświetlić i zmodyfikować zawartość pakietów. Należy zauważyć, że luka ta nie pozwala atakującemu na bezpośrednie wykonanie kodu lub podniesienie poziomu swoich uprawnień. Może zostać wykorzystana do zebrania użytecznych informacji celem dalszego spenetrowania systemu lub sieci, których dotyczy luka.

Niniejsza aktualizacja ma wskaźnik ważności „ważny” dla wszystkich obsługiwanych wersji systemu Windows Vista i Windows Server 2008. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji**.**

Aktualizacja zabezpieczeń usuwa lukę, poprawiając sposób przetwarzania reguł protokołu IPsec. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Firma Microsoft zaleca klientom jak najszybsze zastosowanie tej aktualizacji.

**Znane problemy.** Brak

Luka w zabezpieczeniach zasad protokołu IPsec umożliwiająca ujawnienie informacji — CVE-2008-2246  
-------------------------------------------------------------------------------------------------
  
Luka umożliwiająca ujawnienie informacji występuje w procedurze importowania zasad protokołu IPSec z domen systemu Windows Server 2003 do domen systemu Windows Server 2008. Ta luka może spowodować, że systemy będą ignorować zasady protokołu IPSec i przesyłać dane w sieci w postaci czystego tekstu. To z kolei umożliwia ujawnienie informacji, które powinny być zaszyfrowane podczas przesyłania w sieci. Osoba atakująca przechwytująca ruch sieciowy może wyświetlić i zmodyfikować zawartość pakietów. Należy zauważyć, że luka ta nie pozwala atakującemu na bezpośrednie wykonanie kodu lub podniesienie poziomu swoich uprawnień. Może zostać wykorzystana do zebrania użytecznych informacji celem dalszego spenetrowania systemu, którego dotyczy luka.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2008-2246](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2246).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach zasad protokołu IPsec umożliwiającej ujawnienie informacji — CVE-2008-2246
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   W systemach Windows Vista i Windows Server 2008 reguła odpowiedzi domyślnej nie jest automatycznie włączana. Reguła odpowiedzi domyślnej jest automatycznie włączana tylko podczas tworzenia zasady protokołu IPsec w systemach Windows starszych niż Windows Vista i Windows Server 2008.
  
#### Obejścia dotyczące luki w zabezpieczeniach zasad protokołu IPsec umożliwiającej ujawnienie informacji — CVE-2008-2246
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   Nie należy wybierać opcji „Reguła odpowiedzi domyślnej” podczas tworzenia zasad protokołu IPSec lub usuwać zaznaczenia tej reguły w istniejących zasadach, ponieważ reguła ta nie jest już używana w systemach Windows Vista ani Windows Server 2008 i jest uwzględniana tylko w starszych wersjach systemu Windows. Aby emulować działanie tej reguły w systemach Windows Vista i Windows Server 2008, zobacz [artykuł 942964 z bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/942964/).
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach zasad protokołu IPsec umożliwiającej ujawnienie informacji — CVE-2008-2246
  
**Jaki zakres obejmuje ta luka?**      
Ta luka może doprowadzić do ujawnienia informacji. Osoba atakująca, której uda się wykorzystać tę lukę, może odczytać zawartość pakietów sieciowych (w innym przypadku zawsze zaszyfrowanych). Należy zauważyć, że luka ta nie pozwala atakującemu na bezpośrednie wykonanie kodu lub podniesienie poziomu swoich uprawnień. Może zostać wykorzystana do zebrania użytecznych informacji celem dalszego spenetrowania systemu, którego dotyczy luka.
  
**W jaki sposób powstaje luka?**      
Przyczyną tej luki jest błąd podczas importowania domyślnej zasady protokołu IPsec z domeny systemu Windows Server 2003 do domeny systemu Windows Server 2008. W pewnych warunkach błąd ten powoduje ignorowanie wszystkich reguł protokołu IPSec.
  
**Co to jest IPsec?**      
IPsec to architektura otwartych standardów zapewniania prywatności i bezpiecznej komunikacji w sieciach IP przy użyciu usług zabezpieczeń kryptograficznych. Standardy IPSec są definiowane przez grupę roboczą Internet Engineering Task Force (IETF).
  
IPsec to długoterminowy kierunek rozwoju zabezpieczeń sieci. Zapewnia on dynamiczną ochronę przed atakami na sieci prywatne i internetowe dzięki kompleksowym zabezpieczeniom. Komputery komunikujące się za pośrednictwem protokołu IPSec muszą dysponować jedynie informacjami o nadawcy i odbiorcy. Dzięki protokołowi IPsec można chronić komunikację między grupami roboczymi, komputerami w sieci lokalnej, klientami i serwerami w domenie, oddziałami firm (oddalonymi od siebie fizycznie), sieciami ekstranet oraz mobilnymi klientami. Więcej informacji na temat protokołu IPSec można znaleźć w następującej [witrynie sieci Web firmy Microsoft](http://www.microsoft.com/technet/network/ipsec/ipsecfaq.mspx).
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**      
Osoba atakująca, której uda się wykorzystać tę lukę, może wyświetlić, a w niektórych przypadkach zmodyfikować, pakiety sieciowe przesyłane w postaci czystego tekstu (w innych przypadkach zawsze zaszyfrowane).
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**      
Ta luka może doprowadzić do nieprawidłowego działania funkcji zabezpieczeń. Aby wykorzystać tę lukę, osoba atakująca musi monitorować ruch sieciowy. Następnie może odczytać i spróbować zmodyfikować zawartość pakietu sieciowego przechwyconego w konkretnym punkcie monitorowania. W normalnych warunkach zasady protokołu IPSec chronią ruch sieciowy przed przechwyceniem dzięki szyfrowanej zawartości. Osoba atakująca nie może wykorzystać tej sytuacji, chyba że uprzednio uzyskała uprawnienia administracyjne do kontrolera domeny. Może natomiast spróbować wykorzystać niewiedzę administratora w zakresie konfiguracji zestawu reguł protokołu IPSec, która spowoduje, że przesyłane informacje nie będą szyfrowane.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**      
Szczególnie narażone na skutki tej luki są stacje robocze z systemem Windows Vista i serwery z systemem Windows Server 2008, ponieważ zasady protokołu IPsec znajdujące się w domenie systemu Windows Server 2008 zostały uaktualnione z domeny systemu Windows Server 2003.
  
**W jaki sposób działa aktualizacja?**      
Aktualizacja usuwa lukę, poprawiając sposób przetwarzania reguł protokołu IPSec.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**      
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**      
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.
