Biuletyn zabezpieczeń firmy Microsoft MS08-064 - Ważna
======================================================

Luka w zabezpieczeniach obsługi deskryptora adresów wirtualnych umożliwia podniesienie poziomu uprawnień (956841)
-----------------------------------------------------------------------------------------------------------------

Opublikowano: 14 października 2008 | Zaktualizowano: 15 października 2008

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa lukę w deskryptorze adresów wirtualnych, która została zgłoszona przez użytkowników. Luka ta umożliwia podniesienie poziomu uprawnień, jeśli użytkownik uruchomi specjalnie spreparowaną aplikację. Uwierzytelniona osoba atakująca, której uda się wykorzystać tę lukę, może podnieść poziom uprawnień w systemie, którego dotyczy ten problem. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami administratora.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla wszystkich obsługiwanych wersji systemów Windows XP, Windows Server 2003, Windows Vista i Windows Server 2008. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę, zmieniając sposób, w jaki deskryptor adresów wirtualnych obsługuje zmienne alokacji pamięci. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Firma Microsoft zaleca klientom jak najszybsze zastosowanie tej aktualizacji.

**Znane problemy.** Brak

Luka w zabezpieczeniach deskryptora adresów wirtualnych umożliwiająca podniesienie poziomu uprawnień — CVE-2008-4036  
--------------------------------------------------------------------------------------------------------------------
  
W sposobie obsługi alokacji pamięci menedżera pamięci oraz deskryptorach adresów wirtualnych (VAD, Virtual Address Descriptor) istnieje luka umożliwiająca podniesienie poziomu uprawnień. Luka umożliwia podniesienie poziomu uprawnień, jeśli uwierzytelniona osoba atakująca uruchomi specjalnie spreparowany program w systemie, którego dotyczy problem. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać podniesienie poziomu uprawnień w systemie, w którym występuje luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami administratora.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2008-4036](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4036).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach deskryptora adresów wirtualnych umożliwiającej podniesienie poziomu uprawnień — CVE-2008-4036
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Aby wykorzystać tę lukę, osoba atakująca musi zostać uwierzytelniona.
  
#### Obejścia dotyczące luki w zabezpieczeniach deskryptora adresów wirtualnych umożliwiającej podniesienie poziomu uprawnień — CVE-2008-4036
  
Firma Microsoft nie znalazła żadnego obejścia tej luki.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach deskryptora adresów wirtualnych umożliwiającej podniesienie poziomu uprawnień — CVE-2008-4036
  
**Jaki zakres obejmuje ta luka?**  
Jest to luka umożliwiająca podniesienie uprawnień. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać podniesienie poziomu uprawnień w systemie, w którym występuje luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami administratora.
  
**W jaki sposób powstaje luka?**  
Menedżer pamięci nieprawidłowo obsługuje przepełnienie liczby całkowitej podczas zmniejszania wartości zmiennej, co prowadzi do błędu mapowania podczas alokacji pamięci.
  
**Co to jest deskryptor adresu wirtualnego (VAD)?**  
VAD to forma pamięci wirtualnej dla aplikacji, która umożliwia każdej aplikacji rezerwację własnej prywatnej przestrzeni adresowej. Więcej informacji na temat VAD znajduje się w artykułach MSDN: [Virtual-memory Manager in Windows NT](http://msdn.microsoft.com/en-us/library/ms810616.aspx) i [Managing Virtual memory in Win32](http://msdn.microsoft.com/en-us/library/ms810627.aspx).
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać podniesienie poziomu uprawnień w systemie, w którym występuje luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami administratora.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Uwierzytelniona osoba atakująca może uruchomić specjalnie spreparowany program, który spowoduje przepełnienie liczby całkowitej, a w efekcie — uszkodzenie pamięci w sposób umożliwiający wykonanie kodu.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Szczególnie zagrożone są systemy Windows XP, Windows Server 2003, Windows Vista i Windows Server 2008.
  
**W jaki sposób działa aktualizacja?**  
Przyczyną luki jest przepełnienie liczby całkowitej. Aktualizacja usuwa lukę, dodając procedurę sprawdzania poprawności parametrów VAD.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**  
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**  
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.
