Biuletyn zabezpieczeń firmy Microsoft MS08-059 - Krytyczna
==========================================================

Luka w zabezpieczeniach usługi RPC serwera Host Integration Server umożliwia zdalne wykonanie kodu (956695)
-----------------------------------------------------------------------------------------------------------

Opublikowano: 14 października 2008 | Zaktualizowano: 29 października 2008

**Wersja:** 1.2

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach serwera Microsoft Host Integration Server, która została zgłoszona przez użytkowników. Luka ta umożliwia zdalne wykonanie kodu, gdy osoba atakująca wyśle specjalnie spreparowane żądanie RPC do systemu, którego dotyczy luka. Użytkownicy korzystający z najważniejszych wskazówek, którzy skonfigurują konto usługi SNA RPC tak, aby miało niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, których konto usługi SNA RPC posiada uprawnienia administracyjne.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „krytyczny” dla wszystkich obsługiwanych wersji serwera Microsoft Host Integration Server 2000, Microsoft Host Integration Server 2004 i Microsoft Host Integration Server 2006. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja usuwa lukę, powodując sprawdzanie żądań RPC. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Firma Microsoft zaleca klientom natychmiastowe zastosowanie tej aktualizacji.

**Znane problemy.** W [artykule 956695 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/956695) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń oraz zalecane rozwiązania. Tam, gdzie omówienia znanych problemów i zalecane rozwiązania odnoszą się tylko do określonych wersji tego oprogramowania, w artykule znajdują się łącza do dalszych artykułów.

Luka w zabezpieczeniach usługi HIS związana z wykonaniem polecenia — CVE- 2008-3466  
-----------------------------------------------------------------------------------
  
W usłudze zdalnego wywołania procedury SNA dla serwera Host Integration Server istnieje luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca może wykorzystać tę lukę poprzez utworzenie specjalnie spreparowanego żądania RPC. Luka ta umożliwia zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2008-3466](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3466).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach usługi HIS związanej z wykonaniem polecenia — CVE- 2008-3466
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać takie same uprawnienia, jak uprawnienia konta usługi SNA określone podczas instalacji serwera Host Integration Server. Klienci, którzy zgodnie z najważniejszymi wskazówkami skonfigurują konto usługi SNA RPC tak, aby zmniejszyć jego prawa użytkownika w systemie, mogą być zagrożeni w mniejszym stopniu niż klienci, którzy skonfigurowali konto usługi SNA z prawami administratora.  
-   W przypadku systemów z wersją Microsoft Host Integration Server 2000 Administrator Client lub Microsoft Host Integration Server 2004 Client domyślna instalacja nie powoduje utworzenia ani uruchomienia usługi SNA RPC, w której występuje luka. Pliki są jednak domyślnie instalowane i można je uruchomić, dlatego zdecydowanie zaleca się klientom zastosowanie aktualizacji, jeśli posiadana wersja produktu jest zagrożona przez lukę.
  
#### Obejścia dotyczące luki w zabezpieczeniach usługi HIS związanej z wykonaniem polecenia — CVE- 2008-3466
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   W przypadku programów Host Integration Server 2004 i Host Integration Server 2006 nie konfigurować usługi HIS/SNA do uruchamiania z konta administratora.
  
    -   Podczas pierwszej instalacji serwera Host Integration Server administratorzy są proszeni o wskazanie konta, przy użyciu którego ma być uruchamiana usługa SNA RPC. Klienci powinni postępować zgodnie z najważniejszymi wskazówkami i wybrać konto bez uprawnień administratora.  
    -   Usługę SNA można skonfigurować przy użyciu narzędzia konfiguracji znajdującego się w folderze programu Host Integration Server w menu Start tak, aby wykorzystywała konto o mniejszych uprawnieniach.
  
    **Uwaga** To obejście jest niedostępne dla programu Host Integration Server 2000.
  
    **Wpływ obejścia**. Brak. Zgodnie z najważniejszymi wskazówkami usługę SNA należy uruchamiać przy użyciu konta bez uprawnień administratora.
  
-   W przypadku programów Host Integration Server 2000, Host Integration Server 2004 i Host Integration Server 2006 należy wyłączyć usługę SNA RPC.
  
    1.  Aby wyłączyć usługę, której dotyczy luka, należy ustawić dla usługi SnaRpcService opcję wyłączenia. Usługa ta służy do dodawania liczników wydajności i odświeżania powiązań podczas dodawania do serwera usług połączeń lub dodatkowych usług SNA Server.  
    2.  Po ukończeniu tych zadań konfiguracyjnych należy wyłączyć usługę SNA RPC.
  
    **Wpływ obejścia.** Normalne działanie oraz dodatkowe zmiany konfiguracyjne, takie jak dodawanie połączeń, nie zależą od usługi SNA RPC. Jednak w przypadku dodania usług połączeń lub węzłów w czasie, gdy usługa ta będzie wyłączona, powiązane liczniki wydajności nie zostaną dodane, a usługa SNA MngAgent będzie musiała zostać przetworzona, aby te zmiany w konfiguracji zostały uwzględnione w programie SNA Manager. Wyłączenie usługi SNA RPC nie ma wpływu na zbieranie danych dotyczących wydajności.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach usługi HIS związanej z wykonaniem polecenia — CVE- 2008-3466
  
**Jaki zakres obejmuje ta luka?**    
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami.
  
**W jaki sposób powstaje luka?**    
Specjalnie spreparowane żądania zdalnego wywoływania procedury pozwalają nieuwierzytelnionym zdalnym użytkownikom ominąć uwierzytelnianie w usłudze SNA RPC.
  
**Co to jest RPC?**    
Zdalne wywoływanie procedury (Remote Procedure Call) jest protokołem, za pomocą którego program może zażądać usługi od programu uruchomionego na innym komputerze w sieci. Protokół RPC wspomaga współdziałanie, ponieważ program korzystający z niego nie musi znać protokołów sieciowych obsługujących komunikację. Program żądający usługi za pośrednictwem protokołu RPC jest klientem, natomiast program udostępniający usługi jest serwerem.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem podlegającym luce. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może próbować wykorzystać tę lukę, tworząc specjalnie spreparowany komunikat RPC i wysyłając ten komunikat do systemu jej podlegającego przez port TCP/UDP protokołu RPC, który jest dynamicznie przydzielany przez program Host Integration Server.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Zagrożone są wszystkie systemy, w których zainstalowano dotkniętą omawianym problemem wersję programu Microsoft Host Integration Server.
  
**W jaki sposób działa aktualizacja?**    
Niniejsza aktualizacja usuwa lukę, ograniczając dostęp do funkcji administracyjnych używanych za pośrednictwem usługi RPC.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**    
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.
