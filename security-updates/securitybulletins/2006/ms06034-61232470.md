Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS06-034 - Ważna
======================================================

Luka programu Microsoft Internet Information Services wykorzystująca strony serwera Active Server może pozwolić na zdalne wykonanie kodu (917537)
-------------------------------------------------------------------------------------------------------------------------------------------------

Opublikowano: 11 lipca 2006 | Zaktualizowano: 19 lipca 2006

**Wersja:** 1.2

#### Streszczenie

**Kto powinien przeczytać ten dokument:** Użytkownicy korzystający z programu Microsoft Internet Information Services (IIS)

**Wpływ luki:** Zdalne wykonanie kodu

**Maksymalny wskaźnik ważności:** Ważne

**Zalecenie:** Klienci powinni jak najszybciej zastosować tę aktualizację.

**Zastąpienie aktualizacji zabezpieczeń:** Brak

**Ostrzeżenia:** [W artykule 917537 w bazie wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/917537) udokumentowano znane obecnie problemy, które mogą wystąpić przy instalacji niniejszej aktualizacji zabezpieczeń. Artykuł ten przedstawia również zalecane rozwiązania tych problemów. Więcej informacji na ten temat można znaleźć w [artykule 917537 w bazie wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/917537).

Szczegółowe informacje o luce w zabezpieczeniach
------------------------------------------------

<span></span>
#### Luka w zabezpieczeniach programu Internet Information Services związana z użyciem nieprawidłowo uformowanych stron serwera Active Server — CVE-2006-0026

W programie Internet Information Services (IIS) istnieje luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca może wykorzystać lukę, tworząc specjalnie spreparowany plik stron serwera Active Server. Przetworzenie tego pliku przez program Internet Information Services (IIS) może potencjalnie pozwolić na zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.

#### Czynniki ograniczające zagrożenie spowodowane luką programu Internet Information Services związaną z użyciem nieprawidłowo uformowanych stron serwera Active Server — CVE-2006-0026:

-   W programie IIS 5.0 i IIS 5.1, aplikacje obsługujące strony serwera Active Server są domyślnie uruchamiane w aplikacji „Pooled Out of Process”, co oznacza, że są uruchamiane w pliku DLLHOST.exe, który działa w kontekście konta IWAM\_&lt;nazwakomputera&gt; o niskich uprawnieniach.
-   Domyślnie programy IIS 5.1 w systemie Windows XP Professional i IIS 6.0 w systemie Windows Server 2003 są wyłączone.
-   Strony serwera Active Server są domyślnie wyłączone w IIS 6.0. Po ich włączeniu działają w kontekście procesu roboczego W3WP. uruchomionego jako konto „NetworkService” o niskich uprawnieniach.
-   Aby wykorzystać tę lukę, osoba atakująca musiałaby dysponować prawidłowymi poświadczeniami logowania. Jeśli jednak serwer został specjalnie skonfigurowany tak, aby zezwalać anonimowym lub uwierzytelnionym użytkownikom na przekazywanie treści sieci Web, takich jak strony .ASP, do witryn sieci Web, możliwy jest udany atak na taki serwer z wykorzystaniem omawianej luki.

#### Rozwiązania problemu spowodowanego luką programu Internet Information Services związaną z użyciem nieprawidłowo uformowanych stron serwera Active Server — CVE-2006-0026:

Nie znaleziono żadnego rozwiązania dla tej luki.

#### Często zadawane pytania dotyczące luki programu Internet Information Services związanej z użyciem nieprawidłowo uformowanych stron serwera Active Server — CVE-2006-0026:

**Jaki zakres obejmuje luka?**
W programie Internet Information Services (IIS) istnieje luka umożliwiająca zdalne wykonanie kodu związana ze sposobem obsługi przez ten program stron serwera Active Server (ASP). Osoba atakująca może wykorzystać lukę, tworząc specjalnie spreparowany plik stron serwera Active Server. Przetworzenie tego pliku przez program Internet Information Services (IIS) może potencjalnie pozwolić na zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.

**Skąd bierze się luka?**
Niesprawdzony bufor w programie IIS.

**Co to są strony serwera Active Server?**
Strony serwera Active Server (ASP) to technologia skryptów po stronie serwera, która może służyć do tworzenia dynamicznych i interaktywnych aplikacji sieci Web. Strona ASP jest dokumentem HTML zawierającym skrypty wykonywane po stronie serwera, które przed wysłaniem do przeglądarki użytkownika są przetwarzane przez serwer sieci Web.

**Jakie mogą być skutki wykorzystania luki przez osobę atakującą?**
Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.

**Kto może wykorzystać lukę?**
Aby wykorzystać tę lukę, osoba atakująca musiałaby dysponować prawidłowymi poświadczeniami logowania na serwerze. Jeśli jednak serwer został specjalnie skonfigurowany tak, aby zezwalać anonimowym lub uwierzytelnionym użytkownikom na przekazywanie treści sieci Web, takich jak strony .ASP, do witryn sieci Web, możliwy byłby atak na taki serwer z wykorzystaniem omawianej luki.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**
Osoba atakująca może próbować wykorzystać tę lukę, tworząc specjalnie spreparowany plik ASP i wysyłając ten plik do systemu, którego dotyczy luka. Przetworzenie tego pliku przez program IIS może następnie spowodować wykonanie kodu przez system, którego dotyczy luka.

**Jakie systemy są szczególnie narażone na skutki tej luki?**
Szczególnie narażone na skutki tej luki są systemy Windows 2000 z dodatkiem Service Pack 4 z działającym programem IIS 5.0, ponieważ program IIS jest w nich domyślnie włączony. Zagrożone są również systemy Windows XP Professional i Windows Server 2003, ponieważ jest w nich włączona ta usługa.

**Czy lukę można wykorzystać zdalnie za pomocą Internetu?**
Tak. Osoba atakująca może wykorzystać lukę za pośrednictwem Internetu, jeśli uzyska dostęp do zapisu na serwerze. Wskazówki dotyczące zapór i standardowe domyślne konfiguracje zapór mogą pomóc w ochronie przed atakami z sieci Internet. Firma Microsoft udostępnia informacje mogące pomóc w zabezpieczeniu komputera. Użytkownicy końcowi mogą odwiedzić [witrynę sieci Web Chroń swój komputer](http://www.microsoft.com/poland/athome/security/protect/windowsxpsp2/default.mspx). Specjaliści IT mogą odwiedzić witrynę sieci Web [Security Guidance Center](http://www.microsoft.com/poland/technet/security/) (centrum porad w zakresie bezpieczeństwa).

**Czy luka była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika. Firma Microsoft nie otrzymała żadnych informacji wskazujących na to, że w czasie, gdy niniejszy biuletyn został opublikowany po raz pierwszy, luka ta była znana szerokiemu gronu użytkowników.

**Czy firma Microsoft otrzymała zgłoszenia o wykorzystaniu luki w czasie, gdy opublikowano niniejszy biuletyn?**
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.

**Wymaganie dotyczące ponownego uruchomienia**

Ta aktualizacja nie wymaga ponownego uruchomienia komputera. Instalator zatrzymuje wymagane usługi, instaluje aktualizację i ponownie uruchamia usługi. Jeżeli jednak z jakiegoś powodu nie jest możliwe zatrzymanie wymaganych usług lub jeżeli wymagane pliki są aktualnie używane, ta aktualizacja wymaga ponownego uruchomienia. W takiej sytuacji wyświetlany jest komunikat z zaleceniem ponownego uruchomienia.

Aby zmniejszyć prawdopodobieństwo konieczności ponownego uruchomienia, przed zainstalowaniem aktualizacji zabezpieczeń należy zatrzymać wszystkie usługi, których dotyczy luka, i zamknąć wszystkie aplikacje, które mogą korzystać z plików, których dotyczy luka. Więcej informacji o powodach wyświetlenia monitu o ponowne uruchomienie komputera znajduje się w [artykule 887012 bazy wiedzy Microsoft Knowledge Base (j.ang.)](http://support.microsoft.com/kb/887012).

Ta aktualizacja zabezpieczeń obsługuje funkcję HotPatching w systemach posiadających wersję 6.0.3790.1830 pliku Asp.dll. HotPatching umożliwia instalowanie aktualizacji zabezpieczeń na komputerach z systemem Microsoft Windows Server 2003 z dodatkiem Service Pack 1 bez konieczności ich ponownego uruchomienia. Więcej informacji o funkcji HotPatching i sposobie jej wykorzystania do wdrażania aktualizacji zabezpieczeń można znaleźć w [artykule 897341 bazy wiedzy Microsoft Knowledge Base (j.ang.)](http://support.microsoft.com/kb/897341).

Aktualizacja dostępna w witrynach Windows Update i Microsoft Update oraz za pośrednictwem funkcji Aktualizacje automatyczne obsługuje funkcję HotPatching i po jej zainstalowaniu nie jest konieczne ponowne uruchomienie serwera.
