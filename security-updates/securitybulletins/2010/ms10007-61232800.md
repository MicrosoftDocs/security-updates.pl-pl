Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS10-007 - Krytyczna
==========================================================

Luka w zabezpieczeniach procedury obsługi powłoki systemu Windows umożliwia zdalne wykonanie kodu (975713)
----------------------------------------------------------------------------------------------------------

Opublikowano: 9 lutego 2010

**Wersja:** 1.0

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń usuwa lukę w systemach Microsoft Windows 2000, Windows XP i Windows Server 2003, która została zgłoszona przez użytkowników. Nie dotyczy ona innych wersji systemu Windows. Luka ta może pozwolić na zdalne wykonanie kodu, jeśli aplikacja, taka jak przeglądarka sieci Web, przekaże specjalnie spreparowane dane do funkcji API ShellExecute za pośrednictwem procedury obsługi powłoki systemu Windows.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „krytyczny” dla wszystkich obsługiwanych wersji systemów Microsoft Windows 2000, Windows XP i Windows Server 2003. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Niniejsza aktualizacja zabezpieczeń usuwa lukę, zmieniając na poprawny sposób, w jaki funkcja API ShellExecute sprawdza poprawność parametrów wejściowych. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją bezzwłocznie przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

Luka w zabezpieczeniach związana ze sprawdzaniem poprawności adresów URL - CVE-2010-0027  
----------------------------------------------------------------------------------------
  
<span></span>
W wersjach systemu Microsoft Windows dotkniętych przez omawiany problem istnieje luka w zabezpieczeniach umożliwiająca zdalne wykonanie kodu. Jej występowanie wynika z nieprawidłowego sprawdzania poprawności danych wejściowych przesyłanych do funkcji API ShellExecute. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2010-0027](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0027).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach związanej ze sprawdzaniem poprawności adresów URL - CVE-2010-0027
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Osoba atakująca, której uda się wykorzystać tę lukę, może uruchomić dowolny kod jako użytkownik zalogowany. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca może uzyskać pełną kontrolę nad systemem podlegającym luce. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.
  
#### Obejścia problemu dotyczącego luki w zabezpieczeniach związanej ze sprawdzaniem poprawności adresów URL - CVE-2010-0027
  
Firma Microsoft nie znalazła żadnego obejścia tej luki.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach związanej ze sprawdzaniem poprawności adresów URL - CVE-2010-0027
  
**Jaki zakres obejmuje ta luka?**  
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może wykonywać działania z takimi samymi prawami, jak użytkownik, który jest aktualnie zalogowany.
  
**W jaki sposób powstaje luka?**  
Gdy aplikacja, taka jak przeglądarka sieci Web, korzysta z funkcji API ShellExecute w celu przetworzenia specjalnie spreparowanych danych, to może ona niewłaściwie sprawdzić poprawność strumienia danych i wykonać plik binarny z lokalnego systemu klienta.
  
**Co to jest powłoka Windows?**  
Interfejs użytkownika (UI) systemu Microsoft Windows zapewnia użytkownikowi dostęp do wielu różnych obiektów niezbędnych do uruchamiania aplikacji i zarządzania systemem operacyjnym. Najliczniejszą i najbardziej znaną grupę tych obiektów stanowią foldery i pliki znajdujące się na dyskach komputerów. Istnieje również wiele obiektów wirtualnych, które umożliwiają użytkownikowi wykonywanie zadań, takich jak wysyłanie plików do zdalnych drukarek lub dostęp do Kosza. Powłoka organizuje te obiekty w hierarchiczną przestrzeń nazw, zapewniając użytkownikowi oraz aplikacjom konsekwentny i efektywny sposób dostępu do obiektów oraz zarządzania nimi.
  
**Co to jest funkcja ShellExecute?**  
ShellExecute to jedna z funkcji interfejsu programowania aplikacji (API) powłoki systemu Windows. Wykonuje ona działania na określonym pliku. Może to na przykład być wywołanie właściwej procedury obsługi dla danego typu pliku. Więcej informacji na temat funkcji ShellExecute można znaleźć w artykule sieci MSDN [Funkcja ShellExecute](http://msdn.microsoft.com/en-us/library/bb762153(vs.85).aspx).
  
**Co to jest interfejs programowania aplikacji (API)?**  
API to zestaw funkcji pozwalających wykorzystywać składniki, aplikacje lub system operacyjny. Interfejs API zazwyczaj składa się z jednej lub wielu bibliotek DLL oferujących określone zestawy funkcji.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**  
Osoba atakująca, której uda się wykorzystać tę lukę, może uruchomić dowolny kod jako użytkownik zalogowany. Jeśli użytkownik jest zalogowany jako administrator, osoba atakująca może uzyskać pełną kontrolę nad systemem podlegającym luce. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Użytkownicy, których konta zostały skonfigurowane w taki sposób, że mają niewielkie uprawnienia w systemie, ponoszą mniejsze ryzyko niż ci, którzy pracują z uprawnieniami administracyjnymi.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**  
Osoba atakująca może dostarczyć zawartość, taką jak specjalnie spreparowana strona sieci Web, która spowodowałaby przeanalizowanie przez aplikację nieprawidłowo utworzonego ciągu. Po przekazaniu tej zawartości do funkcji API ShellExecute, na przykład w formie hiperłącza, osoba atakująca zyskałaby możliwość wykonania pliku binarnego z systemu klienta bez wiedzy zalogowanego użytkownika.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**  
Na skutki tej luki, która może zostać wykorzystana w kontekście zalogowanego użytkownika, szczególnie narażone są systemy klienckie, których używa się do przeglądania niezaufanej zawartości z Internetu.
  
**W jaki sposób działa aktualizacja?**  
Aktualizacja zabezpieczeń usuwa lukę, korygując sposób, w jaki funkcja API ShellExecute sprawdza poprawność parametrów wejściowych w obsługiwanych wersjach systemów Microsoft Windows 2000, Windows XP i Windows Server 2003.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**  
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**  
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.
