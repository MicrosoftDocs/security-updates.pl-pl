Security Bulletin

Biuletyn zabezpieczeń firmy Microsoft MS10-025 - Krytyczna
==========================================================

Luka w zabezpieczeniach programu Microsoft Windows Media Services może umożliwić zdalne wykonanie kodu (980858)
---------------------------------------------------------------------------------------------------------------

Opublikowano: 13 kwietnia 2010 | Zaktualizowano: 27 kwietnia 2010

**Wersja:** 3.0

### Informacje ogólne

#### Streszczenie

Ta aktualizacja zabezpieczeń usuwa lukę w zabezpieczeniach programu Windows Media Services działającego w systemie Microsoft Windows 2000 Server, która została zgłoszona przez użytkowników. Luka ta umożliwia zdalne wykonanie kodu, gdy osoba atakująca wyśle specjalnie spreparowany pakiet informacji transportowych do systemu Microsoft Windows 2000 Server, w którym uruchomiono program Windows Media Services. Sprawdzone metody działania stosowane w zaporach oraz standardowe domyślne konfiguracje zapór mogą pomóc w ochronie sieci przed atakami spoza przedsiębiorstwa. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów. Program Windows Media Services jest opcjonalnym składnikiem systemu Microsoft Windows 2000 Server i domyślnie nie jest instalowany.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „krytyczna” dla wszystkich obsługiwanych edycji systemu Microsoft Windows 2000 Server z uruchomionym programem Windows Media Services. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem, i programy, których nie dotyczy problem** niniejszej sekcji.

Tę aktualizację opublikowano ponownie dnia 27 kwietnia 2010 r. Eliminuje ona lukę w zabezpieczeniach poprzez zmianę sposobu, w jaki usługa Windows Media Unicast Service (nsum.exe) obsługuje pakiety sieciowe informacji transportowych. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją bezzwłocznie przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

Luka w zabezpieczeniach programu Media Services związana z przepełnieniem buforu opartego na stosie - CVE-2010-0478  
-------------------------------------------------------------------------------------------------------------------
  

W systemie Microsoft Windows 2000 Server z dodatkiem Service Pack 4, w którym uruchomiony jest opcjonalny składnik Windows Media Services, istnieje luka w zabezpieczeniach umożliwiająca zdalne wykonanie kodu, wynikająca ze sposobu, w jaki usługa Windows Media Unicast Service obsługuje specjalnie spreparowane pakiety informacji transportowych. Program Windows Media Services jest opcjonalnym składnikiem systemu Microsoft Windows 2000 Server z dodatkiem Service Pack 4 i domyślnie nie jest instalowany. Omawiana luka dotyczy tylko systemów Microsoft Windows 2000 Server, w których uruchomiony jest składnik Windows Media Services.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2010-0478](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0478).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach programu Media Services związanej z przepełnieniem buforu opartego na stosie - CVE-2010-0478
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Opisana w niniejszym biuletynie luka w zabezpieczeniach nie dotyczy obsługiwanych wersji systemów Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008, Windows 7 oraz Windows Server 2008 R2.  
-   Program Windows Media Services domyślnie nie jest włączony w systemie Microsoft Windows 2000 Server. System Microsoft Windows 2000 Server jest zagrożony luką, jeśli zostanie skonfigurowany jako serwer strumieniowego przesyłania mediów poprzez dodanie do niego składnika Windows Media Services w Kreatorze składników systemu Windows.
  
#### Obejścia dotyczące luki w zabezpieczeniach programu Media Services związanej z przepełnieniem buforu opartego na stosie - CVE-2010-0478
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   **Zatrzymanie i wyłączenie usługi Windows Media Unicast Service**
  
    Jako administrator wyłącz usługę Windows Media Unicast Service przy użyciu następującego polecenia w wierszu polecenia:  
    `sc stop nsunicast & sc config nsunicast start= disabled`
  
    **Wpływ obejścia.**  Połączenia z serwerem strumieniowego przesyłania mediów za pośrednictwem usługi Windows Media Unicast Service będą zabronione.
  
    **Cofnięcie obejścia.**
  
    Jako administrator włącz usługę Windows Media Unicast Service przy użyciu następującego polecenia w wierszu polecenia:  
    `sc config nsunicast start= auto & sc start nsunicast`
  
-   **Odinstalowanie składnika Windows Media Services przy użyciu Kreatora składników systemu Windows**
  
    1.  Zaloguj się do komputera jako administrator lub jako członek grupy Administratorzy.  
    2.  Kliknij przycisk **Start**, wskaż polecenie **Ustawienia**, a następnie kliknij polecenie **Panel sterowania**.  
    3.  W Panelu sterowania kliknij dwukrotnie polecenie **Dodaj lub usuń programy.**  
    4.  Kliknij polecenie **Dodaj/Usuń składniki systemu Windows**. Zostanie uruchomiony Kreator składników systemu Windows i zostanie wyświetlony ekran składników systemu Windows.  
    5.  Usuń zaznaczenie pola wyboru **Windows Media Services**. Kliknij przycisk **Dalej** i wykonaj instrukcje Kreatora składników systemu Windows.
  
    **Wpływ obejścia.**  Serwer nie będzie skonfigurowany jako serwer strumieniowego przesyłania mediów korzystający z programu Windows Media Services.
  
    **Jak cofnąć obejście.** Zainstaluj składnik Windows Media Services za pośrednictwem Kreatora składników systemu Windows.
  
#### Często zadawane pytania dotyczące luki w zabezpieczeniach programu Media Services związanej z przepełnieniem buforu opartego na stosie - CVE-2010-0478
  
**Jaki zakres obejmuje ta luka?**    
Jest to luka umożliwiająca zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami.
  
**W jaki sposób powstaje luka?**    
Przyczyną powstawania luki jest sposób obsługi pakietów sieciowych informacji transportowych przez program Windows Media Services uruchomiony w systemie Microsoft Windows 2000 Server z dodatkiem Service Pack 4.
  
**Co to jest Windows Media Services?**    
Microsoft Windows Media Services jest uznaną w branży platformą do strumieniowego przesyłania na żywo lub na żądanie zawartości audio i wideo za pośrednictwem Internetu lub intranetu. Program Windows Media Services umożliwia skonfigurowanie i zarządzanie jednym lub wieloma serwerami Windows Media, które dostarczają zawartość klientom.
  
Klientami mogą być:
  
-   Komputery i urządzenia odtwarzające zawartość przy użyciu odtwarzaczy, takich jak program Windows Media Player.  
-   Komputery z programem Windows Media Services, które służą jako proxy, buforują lub redystrybuują zawartość.  
-   Niestandardowe aplikacje, które powstały przy użyciu zestawu SDK Windows Media Software Development Kit.
  
**Który składnik programu Windows Media Services jest zagrożony przez lukę?**    
Niebezpieczny kod znajduje się w usłudze Windows Media Unicast Service (nsum.exe).
  
**Co to jest usługa Windows Media Unicast Service?**    
Usługa Windows Media Unicast Service to jedna z usług Windows Media Services - grupy usług, które umożliwiają dostawcom zawartości cyfrowej przesyłanie klientom multimediów strumieniowych. Należą do niej cztery usługi:
  
-   Windows Media Unicast Service - umożliwia przesyłanie multimediów strumieniowych do określonego użytkownika końcowego.  
-   Windows Media Station Service - umożliwia jednoczesne przesyłanie wielu użytkownikom końcowym pojedynczego strumienia zawartości multimedialnej.  
-   Windows Media Program Service - kontroluje liczbę odtworzeń grupy strumieni.  
-   Windows Media Monitor Service - umożliwia dostawcom zawartości cyfrowej monitorowanie użytkowników końcowych podłączonych do punktów publikacji.
  
Omawiana luka dotyczy tylko usługi Windows Media Unicast Service. Nie można jej wykorzystać jako zagrożenia dla pozostałych trzech usług.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może wykorzystać tę lukę, wysyłając specjalnie spreparowany pakiet informacji transportowych do komputera z systemem Microsoft Windows 2000 Server z dodatkiem Service Pack 4, w którym uruchomiono program Windows Media Services. W zabezpieczeniu sieci przed atakami pochodzącymi spoza przedsiębiorstwa mogą pomóc wskazówki dotyczące ustawień zapory oraz standardowe domyślne konfiguracje zapory. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów.
  
Program Windows Media Services jest opcjonalnym składnikiem systemu Microsoft Windows 2000 Server z dodatkiem Service Pack 4 i domyślnie nie jest instalowany.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Omawiana luka dotyczy systemu Microsoft Windows 2000 Server z dodatkiem Service Pack 4 z uruchomionym programem Windows Media Services.
  
**W jaki sposób działa aktualizacja?**    
Ta aktualizacja usuwa lukę w zabezpieczeniach, modyfikując sposób, w jaki usługa Windows Media Unicast Service obsługuje pakiety sieciowe informacji transportowych.
  
**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Choć początkowy raport powstał dzięki ujawnieniu luki w sposób odpowiedzialny, to luka ta została później publicznie ujawniona.
  
**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**    
Nie. Firma Microsoft nie otrzymała żadnych informacji o tym, że luka ta była szeroko wykorzystywana do atakowania użytkowników i nie spotkała się z przykładami odpowiednich kodów w czasie, gdy został opublikowany niniejszy biuletyn.

**Instalowanie aktualizacji**

Podczas instalowania tej poprawki zabezpieczeń instalator sprawdza, czy dowolny z aktualizowanych plików na komputerze był wcześniej aktualizowany za pomocą poprawki hotfix firmy Microsoft.

**Weryfikacja instalacji aktualizacji**

-   **Narzędzie Microsoft Baseline Security Analyzer (MBSA)**

    Aby sprawdzić, czy ta aktualizacja zabezpieczeń została zainstalowana w systemie, którego dotyczy luka, można skorzystać z narzędzia Microsoft Baseline Security Analyzer (MBSA). Więcej informacji na ten temat znajduje się w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się powyżej w niniejszym biuletynie.

-   **Weryfikacja wersji pliku**

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego w systemie, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

-   **Weryfikacja klucza rejestru**

    Pliki, które zostały zainstalowane przez tę aktualizację zabezpieczeń, można także zweryfikować, przeglądając klucze rejestru wymienione w niniejszej sekcji w **tabeli informacyjnej**.

    Te klucze rejestru mogą nie zawierać pełnej listy zainstalowanych plików. Klucze te mogą także nie zostać prawidłowo utworzone, jeśli administrator lub producent OEM zintegruje tę aktualizację zabezpieczeń ze źródłowymi plikami instalacyjnymi systemu Windows lub ją do nich włączy.
