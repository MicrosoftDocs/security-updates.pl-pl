Biuletyn zabezpieczeń firmy Microsoft MS09-018 - Krytyczna
==========================================================

Luka w zabezpieczeniach usługi Active Directory umożliwia zdalne wykonanie kodu (971055)
----------------------------------------------------------------------------------------

Opublikowano: 9 czerwca 2009 | Zaktualizowano: 17 czerwca 2009

**Wersja:** 1.1

### Informacje ogólne

#### Streszczenie

Niniejsza aktualizacja zabezpieczeń pozwala usunąć zgłoszone przez użytkowników dwie luki w zabezpieczeniach wdrożeń usługi Active Directory w systemach Microsoft Windows 2000 Server i Windows Server 2003 oraz Trybu aplikacji usługi Active Directory (ADAM) w systemach Windows XP Professional i Windows Server 2003. Poważniejsza luka w zabezpieczeniach potencjalnie umożliwia zdalne wykonanie kodu. Osoba atakująca, której uda się wykorzystać tę lukę, może zdalnie uzyskać pełną kontrolę nad systemem jej podlegającym. Osoba taka może wówczas instalować programy, przeglądać, zmieniać i usuwać dane oraz tworzyć nowe konta z pełnymi uprawnieniami. Sprawdzone metody działania stosowane w zaporach oraz standardowe domyślne konfiguracje zapór mogą pomóc w zabezpieczeniu sieci przed atakami spoza obszaru działania przedsiębiorstwa. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „krytyczny” dla wszystkich obsługiwanych wersji systemu Microsoft Windows 2000 Server oraz wskaźnik ważności „ważny” dla obsługiwanych wersji systemów Windows XP Professional i Windows Server 2003. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Aktualizacja zabezpieczeń usuwa lukę poprzez poprawienie sposobu, w jaki usługa LDAP przydziela i zwalnia pamięć podczas przetwarzania specjalnie spreparowanych żądań LDAP lub LDAPS.

**Zalecenie.** Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją bezzwłocznie przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

Luka w zabezpieczeniach usługi Active Directory związana z nieprawidłowym zwalnianiem pamięci — CVE-2009-1138  
-------------------------------------------------------------------------------------------------------------
  
We wdrożeniach usługi Active Directory w systemie Microsoft Windows 2000 Server występuje luka w zabezpieczeniach umożliwiająca zdalne wykonanie kodu. Przyczyną jej występowania jest nieprawidłowe zwalnianie pamięci w przypadku przetwarzania specjalnie spreparowanego zapytania LDAP lub LDAPS. Osoba atakująca, której uda się wykorzystać tę lukę, może uzyskać pełną kontrolę nad systemem, którego dotyczy luka.
  
Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2009-1138](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1138).
  
#### Czynniki ograniczające zagrożenie wynikające z luki w zabezpieczeniach usługi Active Directory związanej z nieprawidłowym zwalnianiem pamięci — CVE-2009-1138
  
Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:
  
-   Sprawdzone metody działania stosowane w zaporach oraz standardowe domyślne konfiguracje zapór mogą pomóc w zabezpieczeniu sieci przed atakami spoza obszaru działania przedsiębiorstwa. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów.
  
#### Obejścia problemu wynikającego z luki w zabezpieczeniach usługi Active Directory związanej z nieprawidłowym zwalnianiem pamięci — CVE-2009-1138
  
Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:
  
-   **Zablokowanie portów TCP 389, 636, 3268 oraz 3269 na zaporze**  
  
    Zablokowanie na zaporze następujących protokołów i portów:
  
    Porty TCP 389, 636, 3268 i 3269
  
    Te porty są używane do inicjowania połączenia ze składnikiem, którego dotyczy luka. Zablokowanie tych portów na zaporze przedsiębiorstwa zarówno dla ruchu przychodzącego, jak i wychodzącego, może pomóc w zabezpieczeniu systemów chronionych zaporą przed próbami wykorzystania luki opisanej w tym biuletynie. Porty 389 i 636 zapewniają odpowiednio usługę LDAP i bezpieczną usługę LDAP, natomiast porty 3268 i 3269 są używane przez serwer wykazu globalnego, który także przetwarza żądania LDAP.

#### Często zadawane pytania dotyczące luki w zabezpieczeniach usługi Active Directory związanej z przeciekiem pamięci — CVE-2009-1139
  
**Jaki zakres obejmuje ta luka?**    
Ta luka powoduje awarię typu odmowa usługi. Po wykorzystaniu luki przez osobę atakującą zaatakowany system może przestać reagować. Może być wymagane ponowne uruchomienie systemu. Należy zauważyć, że luka umożliwiająca atak typu „odmowa usługi" nie pozwala atakującemu na wykonanie kodu lub podniesienie poziomu swoich uprawnień, lecz może spowodować, że zaatakowany system przestanie akceptować żądania.
  
**W jaki sposób powstaje luka?**    
Usługa LDAP nieprawidłowo zarządza pamięcią podczas wykonywania żądań LDAP lub LDAPS, które zawierają określone filtry identyfikatorów obiektów (OID).
  
**Co to jest LDAP?**    
LDAP (Lightweight Directory Access Protocol) to otwarty standard protokołu internetowego opracowany z myślą o zapewnianiu dostępu do rozproszonych katalogów.
  
**Co to jest LDAP over SSL (LDAPS)?**    
Domyślnie ruch LDAP jest przesyłany w postaci niezabezpieczonej. Możliwe jest jednak utajnienie i zabezpieczenie ruchu LDAP poprzez użycie technologii Secure Sockets Layer (SSL)/Transport Layer Security (TLS). Administratorzy mogą włączyć protokół LDAPS (LDAP over SSL), instalując odpowiednio sformatowany certyfikat od urzędu certyfikacji firmy Microsoft lub innego urzędu certyfikacji. Więcej informacji na ten temat można znaleźć w [artykule 321051 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/321051).
  
**Czy protokół LDAP over SSL jest domyślnie dostępny?**    
Nie. Aby serwer LDAP mógł uczestniczyć w sesji SSL, administrator musi uzyskać cyfrowy certyfikat i zainstalować go na tym serwerze. Jeśli nie zostanie to zrobione, protokół LDAP over SSL pozostanie niedostępny.

netstat –a

**Co to są filtry identyfikatorów obiektów (OID)?**    
Identyfikator obiektu (OID) jest używany do nazywania obiektów LDAP. Zapytanie LDAP może zawierać filtr w celu wybrania z obszaru nazw LDAP tylko tych danych, które są zgodne z określonym identyfikatorem OID.
  
**Co to jest program Tryb aplikacji usługi Active Directory (ADAM)?**    
ADAM to usługa katalogowa LDAP, która nie działa jako usługa systemowa, ale jako usługa użytkownika. Więcej informacji można znaleźć w artykule TechNet [ADAM Concepts (j. ang.)](http://technet2.microsoft.com/windowsserver/en/library/29fb059e-544c-4577-bf7c-ba4b08df48431033.mspx?mfr=true).
  
**Czy omawiana luka dotyczy usługi Active Directory Lightweight Directory Service (AD LDS)?**    
Usługi Active Directory Lightweight Directory Services stanowią niezależny tryb usług Active Directory i zapewniają dedykowane usługi katalogowe dla aplikacji. Usługa AD LDS jest obecnie dostępna tylko dla systemu operacyjnego Microsoft Windows Server 2008 i nie dotyczy jej ta luka w zabezpieczeniach.
  
**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Osoba atakująca, której uda się wykorzystać tę lukę, może spowodować, że zaatakowany system przestanie odpowiadać i będzie wymagać ponownego uruchomienia.
  
**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może próbować wykorzystać tę lukę, wysyłając specjalnie spreparowany pakiet LDAP lub LDAPS do serwera Active Directory lub serwera ADAM. W systemie Microsoft Windows 2000 Server tę lukę może wykorzystać każdy użytkownik anonimowy mający dostęp do sieci docelowej, wysyłając specjalnie spreparowany pakiet sieciowy do systemu, którego dotyczy luka. W systemie Windows Server 2003 lub systemach z zainstalowanym programem ADAM osoba atakująca musi dysponować prawidłowymi poświadczeniami logowania, aby móc wykorzystać omawianą lukę.
  
**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Protokół LDAP jest wymagany przez usługę Active Directory i program ADAM. Dlatego luka istnieje w każdym systemie działającym jako serwer Active Directory lub serwer ADAM.

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.

Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.  
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego w systemie, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.  
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.
