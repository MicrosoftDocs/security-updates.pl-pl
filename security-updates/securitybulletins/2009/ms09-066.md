Biuletyn zabezpieczeń firmy Microsoft MS09-066 - Ważna
======================================================

Luka w zabezpieczeniach usługi Active Directory umożliwia przeprowadzenie ataku typu „odmowa usługi” (973309)
-------------------------------------------------------------------------------------------------------------

Opublikowano: 10 listopada 2009

**Wersja:** 1.0

### Informacje ogólne

#### Streszczenie

Ta aktualizacja zabezpieczeń pozwala usunąć zgłoszoną przez użytkowników lukę w zabezpieczeniach usługi Active Directory, trybu aplikacji usługi Active Directory (ADAM) oraz usługi Active Directory Lightweight Directory Service (AD LDS). Luka umożliwia przeprowadzenie ataku typu „odmowa usługi”, jeśli podczas wykonywania określonych typów żądań LDAP lub LDAPS zostanie wyczerpane miejsce na stosie. Ta luka w zabezpieczeniach dotyczy tylko kontrolerów domeny i systemów skonfigurowanych do uruchamiania programu ADAM lub usługi AD LDS.

Niniejsza aktualizacja zabezpieczeń ma wskaźnik ważności „ważny” dla usługi Active Directory, programu ADAM i usługi AD LDS we wszystkich obsługiwanych wersjach systemów Microsoft Windows 2000 Server, Windows XP, Windows Server 2003 i Windows Server 2008. Więcej informacji można znaleźć w podsekcji **Programy, których dotyczy problem i programy, których nie dotyczy problem** niniejszej sekcji.

Ta aktualizacja zabezpieczeń usuwa lukę, modyfikując sposób, w jaki usługa Active Directory, program ADAM i usługa AD LDS przetwarzają nieprawidłowe żądania LDAP i LDAPS. Więcej informacji na temat luki można znaleźć w podsekcji „Często zadawane pytania” w części poświęconej tej luce w następnej sekcji, **Informacje o luce w zabezpieczeniach**.

**Zalecenie.**  Większość użytkowników ma włączoną funkcję automatycznych aktualizacji i nie będzie musiała podejmować żadnych działań, ponieważ niniejsza aktualizacja zabezpieczeń zostanie pobrana i zainstalowana automatycznie. Klienci, którzy nie włączyli funkcji automatycznej aktualizacji, muszą sami sprawdzać ich dostępność i ręcznie zainstalować niniejszą aktualizację. Aby uzyskać informacje na temat określonych opcji konfiguracji w przypadku aktualizowania automatycznego, patrz [artykuł 294871 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Firma Microsoft zaleca administratorom (w przypadku instalacji w przedsiębiorstwach) oraz użytkownikom końcowym, którzy chcą zainstalować tę aktualizację zabezpieczeń ręcznie, aby zastosowali ją jak najszybciej przy użyciu oprogramowania do zarządzania aktualizacjami zabezpieczeń lub poprzez sprawdzenie dostępności aktualizacji przy użyciu usługi [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Informacje na ten temat znajdują się także w sekcji **Narzędzia wykrywania i wdrażania oraz wskazówki**, która znajduje się w dalszej części niniejszego biuletynu.

Luka w zabezpieczeniach usługi LSASS związana z cyklicznym przepełnieniem stosu — CVE-2009-1928
-----------------------------------------------------------------------------------------------

We wdrożeniach usług Active Directory w systemach Microsoft Windows 2000 Server, Windows Server 2003 i Windows Server 2008 istnieje luka umożliwiająca atak typu „odmowa usługi”. Luka istnieje także we wdrożeniach programu Tryb aplikacji usługi Active Directory (ADAM) w systemach Windows XP i Windows Server 2003 oraz usługi Active Directory Lightweight Directory Service (AD LDS) w systemie Windows Server 2008. Obecność luki w zabezpieczeniach wynika z wyczerpania miejsca na stosie podczas wykonywania określonych typów żądań LDAP lub LDAPS. Po udanym wykorzystaniu luki przez osobę atakującą zaatakowany system mógłby przestać reagować.

Aby wyświetlić tę lukę w zabezpieczeniach jako standardową pozycję na liście znanych luk i zagrożeń, zobacz [CVE-2009-1928](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1928).

#### Czynniki ograniczające zagrożenie w przypadku luki w zabezpieczeniach usługi LSASS związanej z cyklicznym przepełnieniem stosu — CVE-2009-1928

Ograniczenie sprowadza się do odpowiedniego ustawienia, typowej konfiguracji lub najważniejszej wskazówki ogólnej, które istnieją w stanie domyślnym i mogą zmniejszyć zagrożenie wykorzystaniem omawianej luki w zabezpieczeniach. W tej sytuacji pomocne mogą być następujące czynniki ograniczające zagrożenie:

-   Sprawdzone metody działania stosowane w zaporach oraz standardowe domyślne konfiguracje zapór mogą pomóc w zabezpieczeniu sieci przed atakami spoza obszaru działania przedsiębiorstwa. Zgodnie ze standardowymi zasadami działania zaleca się, aby w systemach połączonych z Internetem była otwarta jak najmniejsza liczba portów.

#### Obejścia problemu spowodowanego przez lukę w zabezpieczeniach usługi LSASS związaną z cyklicznym przepełnieniem stosu — CVE-2009-1928

Obejście oznacza wprowadzenie zmiany ustawienia lub konfiguracji, która nie powoduje wyeliminowania samej luki, lecz może pomóc w zablokowaniu znanych kierunków ataku przed zastosowaniem aktualizacji Firma Microsoft przetestowała następujące obejścia i przedstawią swoją opinię w kwestii ograniczania przez nie funkcjonalności:

-   **Zablokowanie portów TCP 389, 636, 3268 oraz 3269 na zaporze**  

    Zablokowanie na zaporze następujących protokołów i portów:

    Porty TCP 389, 636, 3268 i 3269

    Te porty są używane do inicjowania połączenia ze składnikiem, którego dotyczy luka. Zablokowanie tych portów na zaporze przedsiębiorstwa zarówno dla ruchu przychodzącego, jak i wychodzącego, może pomóc w zabezpieczeniu systemów chronionych zaporą przed próbami wykorzystania luki opisanej w tym biuletynie. Porty 389 i 636 zapewniają odpowiednio usługę LDAP i bezpieczną usługę LDAP, natomiast porty 3268 i 3269 są używane przez serwer wykazu globalnego, który także przetwarza żądania LDAP. Tryb aplikacji Active Directory (ADAM) oraz usługa Active Directory Lightweight Directory Service (AD LDS) umożliwiają administratorom skonfigurowanie portów LDAP innych niż porty domyślne. W przypadku zastosowania tego obejścia należy zablokować te porty także w zaporze, aby uniknąć wykorzystania luki.

    W celu zapobieżenia atakom mogącym wykorzystywać inne porty firma Microsoft zaleca blokowanie całego niepożądanego ruchu przychodzącego z Internetu. Więcej informacji na temat portów znajduje się w dodatku [Przypisywanie portów TCP i UDP](http://go.microsoft.com/fwlink/?linkid=21312) (j. ang.). Aby uzyskać więcej informacji na temat zapory systemu Windows, zobacz artykuł [Jak skonfigurować zaporę systemu Windows na pojedynczym komputerze](http://www.microsoft.com/technet/security/smallbusiness/prodtech/windowsxp/cfgfwall.mspx).

#### Często zadawane pytania dotyczące luki w zabezpieczeniach usługi LSASS związanej z cyklicznym przepełnieniem stosu — CVE-2009-1928

**Jaki zakres obejmuje ta luka?**    
Ta luka powoduje awarię typu odmowa usługi. Po wykorzystaniu luki przez osobę atakującą zaatakowany system może przestać reagować. Może być wymagane ponowne uruchomienie systemu. Należy zauważyć, że luka umożliwiająca atak typu „odmowa usługi" nie pozwala atakującemu na wykonanie kodu lub podniesienie poziomu swoich uprawnień, lecz może spowodować, że zaatakowany system przestanie akceptować żądania.

**W jaki sposób powstaje luka?**    
Usługa LDAP nieprawidłowo przetwarza określone żądania LDAP i LDAPS w sposób, który może prowadzić do wyczerpania miejsca na stosie.

**Co to jest usługa Active Directory?**    
Głównym zadaniem usługi Active Directory jest dostarczanie centralnych usług uwierzytelniania i autoryzacji dla komputerów z systemem Windows.

**Co to jest usługa Active Directory Lightweight Directory Service?**    
Usługa Active Directory Lightweight Directory Service stanowi niezależny tryb usług Active Directory i zapewnia dedykowane usługi katalogowe dla aplikacji. Usługa AD LDS dostępna jest w systemach Windows Server 2008 lub nowszych i zastępuje tryb aplikacji usługi Active Directory (ADAM), który był dostępny w systemach Windows XP i Windows Server 2003.

**Co to jest LDAP?**    
LDAP (Lightweight Directory Access Protocol) to otwarty standard protokołu internetowego opracowany z myślą o zapewnianiu dostępu do rozproszonych katalogów.

**Co to jest LDAP over SSL (LDAPS)?**    
Domyślnie ruch LDAP jest przesyłany w postaci niezabezpieczonej. Możliwe jest jednak utajnienie ruchu LDAP i zabezpieczenie go przed modyfikacją poprzez użycie technologii Secure Sockets Layer (SSL)/Transport Layer Security (TLS). Administratorzy mogą włączyć protokół LDAPS (LDAP over SSL), instalując odpowiednio sformatowany certyfikat cyfrowy od urzędu certyfikacji firmy Microsoft lub innego urzędu certyfikacji, postępując zgodnie ze wskazówkami zawartymi w [artykule 321051 bazy wiedzy Microsoft Knowledge Base](http://support.microsoft.com/kb/321051).

**Czy protokół LDAP over SSL jest domyślnie dostępny?**    
Nie. Aby serwer LDAP mógł uczestniczyć w sesji SSL, administrator musi uzyskać cyfrowy certyfikat i zainstalować go na tym serwerze. Jeśli nie zostanie to zrobione, protokół LDAP over SSL pozostanie niedostępny.

**Jak można stwierdzić, czy na serwerze działa usługa LDAP, jeśli wykorzystywana jest usługa Active Directory**?  
Aby stwierdzić, czy serwer nasłuchuje zapytań LDAP lub LDAPS, uruchom następujące polecenie z wiersza polecenia administratora i sprawdź, czy ma miejsce nasłuchiwanie przez system na porcie LDAP (389) lub LDAPS (636) bądź na jednym z portów usługi wykazu globalnego (3268 lub 3269):
netstat –a
Wyniki zawierające dowolne z następujących elementów oznaczają, że protokół LDAP jest włączony:
Proto Adres lokalny Adres obcy Stan
TCP 0.0.0.0:389 0.0.0.0:0 NASŁUCHIWANIE
TCP 0.0.0.0:636 0.0.0.0:0 NASŁUCHIWANIE
TCP 0.0.0.0:3268 0.0.0.0:0 NASŁUCHIWANIE
TCP 0.0.0.0:3269 0.0.0.0:0 NASŁUCHIWANIE

**Jakie mogą być skutki wykorzystania tej luki przez osobę atakującą?**    
Osoba atakująca, której uda się wykorzystać tę lukę, może spowodować, że komputer użytkownika przestanie odpowiadać i będzie wymagać ponownego uruchomienia.

**W jaki sposób osoba atakująca może wykorzystać tę lukę?**    
Osoba atakująca może próbować wykorzystać tę lukę, wysyłając specjalnie spreparowany pakiet LDAP lub LDAPS do programu ADAM, usługi AD LDS lub serwera Active Directory. W systemie Microsoft Windows 2000 Server tę lukę może wykorzystać użytkownik anonimowy mający dostęp do sieci docelowej, wysyłając specjalnie spreparowany pakiet sieciowy do systemu, którego dotyczy luka. Na wszystkich platformach zagrożonych luką osoba atakująca musi posiadać prawidłowe poświadczenia uwierzytelniania, aby wykorzystanie luki było możliwe.

**Jakie systemy są szczególnie narażone na skutki tej luki?**    
Protokół LDAP jest wymagany przez usługę Active Directory, program ADAM i usługę AD LDS. Dlatego luka istnieje w każdym systemie działającym jako serwer ADAM, serwer AD LDS lub serwer Active Directory.

**W jaki sposób działa aktualizacja?**    
Aktualizacja usuwa lukę, zapewniając prawidłowe przetwarzanie nieprawidłowych żądań LDAP i LDAPS.

**Czy luka ta była powszechnie znana w czasie, gdy opublikowano niniejszy biuletyn?**    
Nie. Firma Microsoft otrzymała informacje o tej luce od odpowiedzialnego użytkownika.

**Czy przed wydaniem niniejszego biuletynu firma Microsoft otrzymała jakiekolwiek zgłoszenia dotyczące wykorzystania tej luki?**    
Nie. Firma Microsoft przed opublikowaniem niniejszego biuletynu zabezpieczeń nie otrzymała informacji, które wskazywałyby, że luka została publicznie wykorzystana do ataku na systemy klientów.

    Ze względu na to, że dostępne są różne wydania systemu Microsoft Windows, czynności do wykonania na określonym komputerze mogą być inne niż opisane poniżej. W takiej sytuacji należy wykonać odpowiednie kroki, korzystając z dokumentacji dotyczącej produktu.

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.
    2.  W obszarze **Co chcesz wyszukać** okienka **Pomocnik wyszukiwania** kliknij pozycję **Wszystkie pliki i foldery**.
    3.  W polu **Cała nazwa pliku lub jej część** wpisz nazwę pliku z odpowiedniej tabeli zawierającej informacje o pliku i kliknij przycisk **Wyszukaj**.
    4.  Na liście plików kliknij prawym przyciskiem myszy nazwę pliku z odpowiedniej tabeli zawierającej nazwy plików, a następnie kliknij polecenie **Właściwości**.  
        **Uwaga** W niektórych wersjach systemu operacyjnego lub w zależności od programów zainstalowanych w systemie niektóre pliki wymienione w tabeli zawierającej informacje o plikach mogą nie być zainstalowane.
    5.  Na karcie **Wersja** sprawdź wersję pliku zainstalowanego w systemie, porównując ją z wersją wymienioną w odpowiedniej tabeli zawierającej informacje o plikach.  
        **Uwaga** Wszystkie atrybuty poza wersją pliku mogą ulec zmianie podczas instalacji. Porównanie innych atrybutów plików z atrybutami z tabeli zawierającej informacje o plikach nie jest zalecaną metodą sprawdzania instalacji aktualizacji. W niektórych przypadkach podczas instalacji mogą być także zmieniane nazwy plików. Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.

-   **Weryfikacja klucza rejestru**

    1.  Kliknij przycisk **Start**, a następnie wskaż polecenie **Wyszukaj**.

Jeżeli brakuje informacji o pliku lub wersji, należy użyć jednej z dostępnych metod weryfikacji instalacji aktualizacji.
